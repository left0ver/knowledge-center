# 神经网络

神经网络中有一个得分函数，类似$f（x，W）= Wx +b$ ,其中 x 为特征，W 为每个特征对应的权重，b 是每个特征对应的偏执项（用于微调）

> 其中 x ，W ，b 都是矩阵



再根据通过 Loss 函数将得分转为对应的损失（通常损失函数包括数据损失 + 正则惩罚项组成）

> 损失函数需要自己定义，这里的只是一个例子

## 对于分类任务

#### 正向传播

我们通过得分函数计算出对应的得分值，再通过 SoftMax 分类器 将每个得分值转化为对应的概率

> Softmax 分类器主要做的事情： 
>
>- 将通过 $e^x$ 函数将对应的得分值扩大，让其差距更明显
>- 再将上一步得到的结果进行一个归一化处理，即每个类别当前的得分/总得分，从而得出属于每个类别的概率

<img src="https://img.leftover.cn/img-md/202410131159462.png" alt="image-20241013115928392" style="zoom:33%;" />

通过 Softmax 分类器计算出概率之后，将概率值带入到损失函数中，计算出对应的损失（对于分类任务，通常选用对数函数作为损失函数）

#### 反向传播

反向传播的核心在于 在每一层在计算**损失函数对每个权重参数的偏导数（梯度）。**反向传播使用链式法则来逐层计算梯度



#### 神经网络的整体架构

所谓**非线性**是指每经过一层  $* W_i$ 之后，需要迭加一个**非线性函数（即激活函数）**

<img src="https://img.leftover.cn/img-md/202410140948560.png" alt="image-20241014094841504" style="zoom:33%;" />



#### 激活函数

常见的激活函数有 `Sigmoid 函数` ，`Relu函数` ,`Tanh`

- **Sgimoid 激活函数的梯度消失的问题**

  如图所示，当 x 很小/x 很大时，其求导之后的结果`基本为0`(即梯度约为 0)，此时会导致网络的**前层权重更新非常缓慢**，网络无法学习到有效的特征。例如 W1 ,W2,W3,W4 ，当 W3 的梯度约为 0 时，此时 W2、W1 可能就只更新一点点

  

  <img src="https://img.leftover.cn/img-md/202410140952894.png" alt="image-20241014095222838" style="zoom:50%;" />

> `Sigmoid 函数` ,`Tanh函数` 都有梯度消失和梯度爆炸的问题，因此我们一般选择 Relu 函数，其导数要么是 0，要么是 1；或者选择它的一些变体：`Leaky ReLU`、`Parametric ReLU` 、 `Exponential Linear Unit（ELU）`

ReLU 公式: $ max \{0,x\}$

#### 梯度爆炸和梯度消失问题

1. 梯度消失：

     在神经网络中，梯度在逐层反向传播时可能会变得非常小（即导数趋近于 0），导致前面几层的权重更新速度非常慢，网络难以有效训练。使用适当的激活函数（如 ReLU）和权重初始化可以缓解梯度消失问题。

2. 梯度爆炸

     在深层神经网络中（特别是递归神经网络 RNN），反向传播时会逐层计算梯度的累积乘积，如果层数很多，梯度的值会迅速变大，导致梯度爆炸。从而导致网络的权重更新幅度过大，使得参数出现异常，影响模型稳定性。

   **梯度裁剪（Gradient Clipping）可以用于限制梯度的大小，避免梯度爆炸。**

> 梯度裁剪的基本思想：当梯度的大小超过某个阈值时，对梯度进行缩放，使其大小保持在这个阈值以内，从而限制梯度的大小

#### 神经元的个数对结果的影响

神经元个数要适中，太多容易导致过拟合，太少容易导致欠拟合

> 一般使用 64，128，256……

#### 正则化

正则化参数也要适中，太小可能会导致正则化效果不明显，模型可能过拟合，太大可能会导致欠拟合

#### 神经网络过拟合解决办法

1. 神经网络基本都要对数据做`标准化的处理`
2. 正则化

3. Drop-OUT

   在`训练的时候`，每次迭代都会随机杀死一些神经元，这些杀死的神经元在这一次的迭代不会参与更新，从而降低神经网络的强度（如图所示）

# 卷积神经网络



### 卷积涉及的参数

- **卷积核的大小**

  一般为 3*3*x ,N 卡对这个有优化,x 为通道数

- **滑动窗口步长**

  步长越小，则看的越细致，提取到的特征越多

  ![image-20241015160937448](https://img.leftover.cn/img-md/202410151609562.png)

- **边界填充**

  一般在边界填充 0，以平衡边界和中心被卷积的次数，让网络能够更公平地对待边界和中心的点

- **卷积核的个数**

  卷积核的个数决定了**卷积层输出特征图的通道数**
  
  卷积层的卷积核是用来提取输入数据的不同特征的。卷积核的个数越多，意味着卷积层可以提取出更多种类的特征
  
  每个卷积核都会在输入特征图上进行滑动卷积操作，生成一个特征图。因此，如果卷积核的个数为 N，那么卷积层的**输出特征图**的通道数也就是 N。

> 一般卷积核大小会设置为 3，padding 设置为 1（为了保证经过卷积之后输出的特征图的长宽不变），stride 为 1 ，通常两次卷积之后进行一次池化操作（一般情况会选择最大池化,一般情况下是让特征图的长宽减半，因此一般设置 kernel_size =2）

```python

# 定义自己的网络
class My_CNN_Net(nn.Module):
  def __init__(self, *args, **kwargs):
    super().__init__(*args, **kwargs)
    self.conv1 = nn.Sequential(
      # in_channels 表示输入的通道数，
      # out_channels表示输出的通道数，
      # kernel_size为卷积核的大小，大多数情况为3
      # stride 为步长，一般为1
      # padding 填充的宽度，一般需要根据kernel_size的大小来设置，一般情况下会根据公式计算输出的长宽，一般会保持长宽一致，例如kernel_size为3，则padding =1，kernel_size为5，则padding =2，
      nn.Conv2d(in_channels=1, out_channels=16, kernel_size=3, stride=1, padding=1),
      nn.ReLU(),
      # 池化操作压缩特征图，一般情况下是让特征图的长宽减半，因此一般设置kernel_size =2
      nn.MaxPool2d(kernel_size=2)
    )

    self.conv2 = nn.Sequential(
      nn.Conv2d(in_channels=16, out_channels=32, kernel_size=3, stride=1, padding=1),
      nn.ReLU(),
      nn.Conv2d(in_channels=32, out_channels=64, kernel_size=3, stride=1, padding=1),
      nn.ReLU(),
      nn.MaxPool2d(kernel_size=2)
    )
    self.conv3 = nn.Sequential(
      nn.Conv2d(in_channels=64, out_channels=128, kernel_size=3, stride=1, padding=1),
      nn.ReLU(),
    )

    # channel =64，经过池化之后长宽为7 
    self.out = nn.Linear(128 * 7 * 7, out_features=10)

  def forward(self, x):
    x = self.conv1(x)
    x = self.conv2(x)
    x = self.conv3(x)

    # 全连接层的时候需要将特征展平
    # 该函数将1维及其之后的维度展平，0维是batch_size
    x = torch.flatten(x, start_dim=1)
    # print(x.shape)
    output = self.out(x)
    return output
```



```python
def accuracy(y_pred, y_true):
  pred = torch.max(y_pred.data, 1)[1]
  right_nums = torch.eq(pred, y_true).sum()
  # 返回的是正确的数量 和总数量
  return right_nums, len(y_true)
```

```python
my_cnn_net = My_CNN_Net()
learning_rate = 0.001
optimizer = optim.Adam(my_cnn_net.parameters(), lr=learning_rate)
loss_func = nn.CrossEntropyLoss()

for cur_epoch in range(epochs):
  # 当前epoch对的总数
  train_rights = []

  for batch_idx, (data, target) in enumerate(train_loader):
		# 训练模型
    my_cnn_net.train()
    train_pred = my_cnn_net(data)

    loss = loss_func(train_pred, target)
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
	
    train_accuracy_data = accuracy(train_pred, target)
    train_rights.append(train_accuracy_data)

    # 每100个batch_size进行一次test集的测试
    if batch_idx % 100 == 0:
      # 对模型测试
      my_cnn_net.eval()
      test_rights = []
      for test_data, test_target in test_loader:
        output = my_cnn_net(test_data)
        test_accuracy_data = accuracy(output, test_target)
        test_rights.append(test_accuracy_data)
        
        # 计算正确的个数
      train_rights_total = sum([i[0] for i in train_rights])
      train_num_total = sum([i[1] for i in train_rights])
      test_rights_total = sum([i[0] for i in test_rights])
      test_num_total = sum([i[1] for i in test_rights])
      print(
        f"当前epoch{cur_epoch},[{batch_idx * batch_size}/{len(train_dataset)}],当前损失为{(loss.data):.3f},训练集上的准确率为{(train_rights_total / train_num_total):.3f},测试集上的准确率为{(test_rights_total / test_num_total):.3f}")
      test_rights.clear()
  train_rights.clear()

```

# 计算机视觉-神经网络的训练

### 数据预处理

常见的预处理操作有

- resize：将图片的大小统一
- RandomRotation: 将图片随机旋转一个角度
- crop：对图片进行裁剪（例如 CenterCrop）
- 水平翻转：例如 RandomHorizontalFlip
- 垂直翻转：例如 RandomVerticalFlip
- 调整图片的亮度、对比度、饱和度、色相等：例如 ColorJitter （若模型需要在识别较暗的图片时会需要）
- 转为灰度图：例如 RandomGrayscale（基本不用）



### 如何确定 BatchSize

1. 先选择小的 batch size ，然后逐步增加大小（一般会使用 2 的幂次方）

### 学习率的选择

一般会选择 0.001，然后使用学习率衰减策略来逐渐降低学习率，避免过拟合

### 学习率衰减策略

1. StepLR ：在每个指定的 epoch 之后，学习率按固定的衰减因子衰减

2. ExponentialLR： 学习率按照指数衰减
   $$
   lr = lr * \gamma ^ {epoch}
   $$
   

   ```python
   from torch.optim.lr_scheduler import ExponentialLR
   
   scheduler = ExponentialLR(optimizer, gamma=0.9)
   ```

3. 基于性能的衰减（ReduceLROnPlateau）

   根据验证集的性能（如损失或准确率），如果在若干个 epoch 内没有改善，则降低学习率。这种方法适用于模型在训练过程中的不稳定性。

4. 余弦衰减（CosineAnnealingLR）：学习率在训练过程中按照余弦函数衰减，从初始学习率逐渐减小到最小学习率，然后再回升（可用于周期性训练）

5. 循环学习率（CyclicLR）：学习率在每个 epoch 内循环变化，周期性地增加和减少

6. 组合衰减策略：例如前 5 个 epoch ，lr 从 0 增加到 1，之后又按照余弦退火到衰减策略进行衰减

```python
linear_scheduler = LinearLR(optimizer, start_factor=0.0, end_factor=1.0, total_iters=5)

# 余弦退火：从 0.1 退火到 0，持续 95 个 epoch
```

### 学习率预热（用的不多）

学习率预热（Learning Rate Warmup）是一种在训练深度学习模型时逐步增加学习率的策略，通常在训练的最初几个 epoch 中使用。其目的是为了避免在训练初期使用过大的学习率，从而导致模型不稳定或收敛困难

#### 常见的学习率预热的方法

**为什么要学习率预热？**

- 在训练的初期，我们的参数是随机初始化的，一开始太高的学习率会让模型前期的训练不稳定，因此一开始使用小的学习率可以防止模型`乱学`。让模型的训练更加稳定。

**学习率预热策略：**

1. 线性预热：在训练的前 N 个 epoch 内，逐步增加学习率到预定的学习率
2. 指数预热：在训练的前 N 个 epoch 中，学习率按照指数方式增加

### 参数的初始化策略

1. 随机初始化： 使用小的随机值初始化权重，一般从均匀分布或者正态分布中采样

2. Xavier（Glorot）初始化：权重从均匀分布或正态分布中采样，均值为 0，方差为 $\frac{2}{n_{in}+ n_{out}} $ (n_in 和 n_out 为输入输出节点数)。

   > 适用于激活函数是 sigmoid 或 tanh 的网络，能有效缓解梯度消失问题

   ```python
   torch.nn.init.xavier_normal_
   ```

3. He 初始化: 权重从均匀分布或正态分布中采样，均值为 0，方差为$\frac{2}{n_{in}}$

   > 特别适用于 ReLU 激活函数，能够有效缓解梯度消失和爆炸的问题。

   ```python
   torch.nn.init.kaiming_normal_
   ```

4. Orthogonal 初始化: 权重初始化为正交矩阵

   ```python
   torch.nn.init.orthogonal_
   ```

### 优化器的选择

####  SGD

随机梯度下降

#### Momentum

在经典的梯度下降算法中，每次权重的更新都只是依赖于当前计算的梯度，而没有考虑之前的梯度，而 Momentum 算法则使用指数加权平均计算出累计梯度

##### 推导

$g_t $ 为当前计算的梯度，$\beta $ 为衰减因子，$v_t$ 被称为动量
$$
v_t = \beta v_{t-1} + (1-\beta) \cdot g_t
$$
**更新权重参数**
$$
w_{t+1} = w_t - \eta  \cdot v_t
$$


#### Adagrad

Adagrad 是一种自适应学习率的优化算法，他可以根据每个参数的历史梯度动态调整学习率，使得在稀疏数据下更有效

> 通过累积历史梯度的平方和，自适应地调整每个参数的学习率

##### 推导

$$
G_t = G_{t-1} + g_t^2
$$

**更新权重参数：**

$g_t $ 为第 t 步计算出的梯度，$\varepsilon $ 是一个很小的常数，防止分母为 0
$$
w_{t+1} =w_t -\frac{\eta}{\sqrt{G_t + \varepsilon }} \cdot g_t
$$

####  Adadelta

Adadelta 算法是 Adagrad 算法的一种变体，旨在解决 Adagrad 在训练后期学习率迅速衰减的问题，通过动态调整每次的更新步长，使得参数更新自适应于过去梯度的变化，**Adadelta 不需要手动设置全局学习率**。

> Adadelta 优化器的核心思想是通过过去梯度的历史信息，自适应地调整每次更新的步长，使得参数更新能够动态适应当前的梯度变化。

1. 计算累积梯度平方和的指数加权平均，$\rho$  为衰减系数

$$
E[g^2]_t = \rho \cdot E[g^2]_{t-1} + (1-\rho) \cdot g_t^2
$$

2. 计算更新的步长
   $$
   \bigtriangleup  w_t = - \frac{\sqrt{E[\bigtriangleup w^2]_{t-1} +\varepsilon }}   {E[g^2]_t + \varepsilon }
   $$

3. 更新权重参数
   $$
   w_{t+1} = w_t + \bigtriangleup w_t 
   $$

4. 对更新步长的平方进行指数加权平均
   $$
   E[\bigtriangleup w^2]_t = \rho \cdot E[\bigtriangleup w^2]_{t-1} + (1-\rho) \cdot (\bigtriangleup w_t) ^2
   $$
   

#### RMSprop

在 Adagrad 中，使用累积梯度的平方和来动态调整每个参数的学习率，但不足之处在于累积梯度平方和会持续增大，导致学习率不断减小，可能在训练后期趋于零，使得模型停滞不前。

而 RMSprop 的核心思路是对梯度平方和进行指数加权平均，从而保证 $G_t$ 的值增加速度不会那么快，以防止学习率快速衰减

##### 推导

$\gamma $ 为衰减因子
$$
E[g^2]_t = \gamma \cdot E[g^2]_{t-1} + (1-\gamma) \cdot g_t^2
$$


**更新权重参数：**
$$
w_{t+1} = w_t - \frac{\eta}{\sqrt{E[g^2]_t + \varepsilon } } \cdot  g_t
$$


#### Adam

上面说到 Momentum 是对梯度进行限制，而 Adagrad 和 RMSprop 算法是对学习率进行限制（动态调整学习率），而 Adam 可以说是 Momentum 和 RMSprop 的结合，综合考虑了梯度和学习率



##### 推导

1. 计算当前梯度的一阶矩的指数加权平均
2. 计算当前梯度的二阶矩的指数加权平均，用来动态调整学习率

$$
m_t = \beta_1 \cdot m_{t-1} + (1-\beta_1) \cdot g_t \\

v_t = \beta_2 \cdot v_{t-1} + (1-\beta_2) \cdot g_t^2
$$

3. 偏差矫正

   当 t 值很小的时候，`m_t` 和 `v_t` 很小，例如当 t=1 时，`v_t = （1-\beta_2）\cdot g_1^2` 很小，从而导致学习率很大，使得模型训练不稳定

   > 由于 `m_t` 和 `v_t` 都是基于历史梯度的累积，在训练初期，它们的值会偏向于零。直接使用这些未修正的值来更新参数可能导致学习率失真，影响收敛效果。

   
   $$
   \hat{m_t} =  \frac{m_t}{1-\beta_1^t} \\
    
    \hat {v_t} = \frac{v_t}{1- \beta_2^t}
   $$
   

经过矫正之后，当 t 很大的时候$1-\beta_1^t$  约为 1， 和矫正之前差不多，当 t 很小时，$1-\beta_1^t$ 会比较大，因此$\hat{m_t}$ 不会很小。

4. 更新权重参数
   $$
   w_{t+1} = w_t - \frac{\eta}{\sqrt{\hat{v_t}} + \varepsilon } \cdot \hat{m_t}
   $$

#### AdamW

AdamW 则是在 Adam 的基础上应用了权重衰减（类似于 L2 正则化），主要思路就是在权重更新时 $-\lambda \cdot w_t$  ,从而将权重参数向 0 方向推移，让参数不那么大，减小模型的复杂度，防止过拟合

**权重参数更新**

$$

w_{t+1} = w_t - \frac{\eta}{\sqrt{\hat{v_t}} + \varepsilon } \cdot \hat{m_t} - \lambda \cdot w_t

$$


### Batch Normalize

1. 最初的论文是想用它来减少内部协变量转移，但是`后面有论文指出它可能就是通过在每个小批量里面加入噪音来控制模型复杂度，因此他有正则化的效果`，故 bn 不需要和 dropout 混合使用

   > 只有 batch—Size 足够大的时候，BN 才有用，通常会运用在深层的神经网络

2. 通常对于深层的神经网络，前面层的梯度比较小，导致前面层的收敛会很慢，而后面层的梯度大，收敛快。BN 相当于是对输出做了一个线性变化

   > 对于 FC 层，BN 在特征维做变化，对于 Conv，BN 在通道维度做变化；而 layer normalize 是对每个样本做归一化

3. 使用了 BN 通常可以选择一个比较大的 lr，因此 BN 可以加快收敛速度，但通常不会改变模型的最终精度
4. BN 一般作用在 conv、FC 层之后，激活函数之前

```python
def batch_norm(X, gamma, beta, moving_mean, moving_var, eps, momentum):
    # 推理
    if not torch.is_grad_enabled():
        # eps是一个很小的数，防止除0
        X_hat = (X - moving_mean) / torch.sqrt(moving_var + eps)
    else:
      # 训练阶段
        assert len(X.shape) in (2, 4)
        if len(X.shape) == 2:
            mean = torch.mean(X, dim=0)
            var = torch.var(X, dim=0)
        else:
            mean = torch.mean(X, dim=[0, 2, 3])
            var = torch.var(X, dim=[0, 2, 3])
        X_hat = (X - moving_mean) / torch.sqrt(moving_var + eps)
        moving_mean = momentum * moving_mean + (1 - momentum) * mean
        moving_var = momentum * moving_var + (1 - momentum) * var
    Y = gamma * X_hat + beta
    return Y, moving_mean, moving_var

```



#### 推导

假设当前的 batch 有 m 个样本,下面的公式求均值，方差，并标准化
$$
%求均值%
u_B = \frac{1}{m} \cdot \sum_{i=1}^{m}x_i \\

%求方差%
\sigma_B ^2 = \frac{1}{m} \sum_{i=1}^{m} (x_i-u_B)^2 \\

%进行标准化%

\hat{x_i} = \frac{x_i - u_B}{\sqrt{\sigma_B^2 + \varepsilon }}
$$
上面是将其标准化之后的结果，但是实际过程中，不同任务的最佳分布可能不同，因此我们引入可学习的参数 $\gamma$ 和 $\beta$ ，模型可以根据具体数据自动调整这些参数

**引入可学习的参数进行缩放和偏移**
$$
y_i = \gamma \cdot \hat{x_i} + \beta
$$

#### 训练阶段和推理阶段的行为不同

在训练阶段，是在每个 batch 上计算均值和方差，再进行标准化+ 缩放偏移

在推理阶段，我们使用的是在训练阶段通过指数加权平均累积的全局均值和方差来进行计算



$\alpha$ 一般为 0.1, $u_B$ 和 $\sigma_B ^2 $ 是当前 batch 的均值和方差
$$
u_{moving} = \alpha \cdot u_{moving}  + (1-\alpha ) \cdot u_B \\

\sigma_{moving} ^2 =  \alpha \cdot \sigma_{moving} ^2  + (1-\alpha ) \cdot \sigma_B ^2 \\
$$

#### 使用的时机

对于普通的神经网络 ，BN 一般使用在全连接层之后，激活函数之前

对于 CNN，BN 一般使用在卷积层之后，激活函数之前

dropout 的思路在于增加噪声，增加模型对输入数据的扰动鲁棒，提高模型的泛化能力

## 梯度累加

对于像 transform 架构的模型，他的显存要求比较高，所以有时候我们的设备可能不是很好，显存比较少，这样的话我们就只能设置为 batch =1，但是这样子就相当于 SGD 了，训练效果很差，因此我们通常会使用梯度累加的策略，即计算多个 batch 的梯度，当更新的时候使用这几个 batch 的梯度的平均值。



## pytorch 推理模式和训练模式的区别

1. 推理模式下不会启用 dropout
2. 在训练模式下，BN 层会使用每个 batch 的均值和方差来归一化数据，而在推理模式下 BN 层使用整个训练集的均值和方差来归一化数据，因为在推理的时候，通常没有可用的批次数据，并且模型希望使用训练过程中学到的全局统计信息。
3. 在训练过程中，通常会进行数据增强操作（如随机裁剪、旋转、翻转等），以增强模型的泛化能力。推理时通常不会进行数据增强，而是直接使用原始的测试数据进行预测（可能会做一些 resize 的操作）

```python
    for X, y in train_loader:
        X = X.squeeze(1)
        optimizer.zero_grad()
        # print(X.shape)
        y_hat = model(X)
        l = criterion(y_hat, y)
        l.backward()
        optimizer.step()
        loss_epoch += l.item() * len(y)
        predicted = torch.argmax(y_hat, dim=1)
        total += len(y)
        correct += torch.sum(predicted == y)

    print(f"正确率{(correct / total):.4f}, 平均损失为{loss_epoch / total}")
    
    
    
model.eval()
correct = 0
total = 0
with torch.no_grad():
    for X, y in test_loader:
        X = X.squeeze(1)
        pred = torch.argmax(model(X), dim=1)
        total += len(y)
        correct += torch.sum(pred == y).item()

print(f"Test Accuracy: {100 * correct / total:.2f}%")    
```

## attention 的本质

0. 什么是注意力机制？

假设我们有这样一个下图所示的字典，对应于 key - value 的一个映射，那假设我们给一个 query = 57，如何根据这个 dict 大致推断出 这个 query 对应的 value 是多少？

很显然 57 和 56 、58 更接近，因此他对应的值更可能在 43-48 之间。

<img src="https://img.leftover.cn/img-md/202502271442627.png" alt="image-20250227144211516" style="zoom:33%;" />

1. Nadaraya-Watson 核回归：

$$
f(x) = \sum_{i=1}^n \frac{K (x-x_i)}{\sum_{j=1}^n K (x-x_j)} \cdot y_i
$$

$\sum_{i=1}^n \frac{K\cdot (x-x_i)}{\sum_{j=1}^n K (x-x_j)}$ 这个公式可以求出 q 对应于每一个 k 的一个相似度（权重），越接近则权重越高。

我们可以将其变为一个更通用的一个式子
$$
f(x) = \sum_{i=1}^n \alpha (x,x_i)y_i
$$
其中 $\alpha(x,x_i)$则是对应的权重的求解方式。 

> 如果一个键 xi 越是接近给定的查询 x， 那么分配给这个键对应值 yi 的注意力权重就会越大， 也就“获得了更多的注意力”。



2. 带参数注意力池化

   Nadaraya-Watson 核回归是一个非参数的模型，当 key 比较多时（即数据比较多），这个模型的效果还是挺不错的。同时也可以给他加上一个可学习的参数 w,变为

   
   $$
   \sum_{i=1}^{n} \operatorname{softmax}\left(-\frac{1}{2}\left((x-x_{i}\right)w)^{2}\right) y_{i}
   $$

3. 评分函数

$$
当 K (u) = \frac{1}{2\pi} exp (-\frac{u^2}{2})时； \\

\begin{aligned}
f(x) & =\sum_{i=1} \alpha\left(x, x_{i}\right) y_{i} \\
& =\sum_{i=1}^{n} \frac{\exp \left(-\frac{1}{2}\left(x-x_{i}\right)^{2}\right)}{\sum_{j=1}^{n} \exp \left(-\frac{1}{2}\left(x-x_{j}\right)^{2}\right)} y_{i} \\
& =\sum_{i=1}^{n} \operatorname{softmax}\left(-\frac{1}{2}\left(x-x_{i}\right)^{2}\right) y_{i}
\end{aligned}
$$

这个 K（u）可以被称为注意力评分函数，再将计算出的得分输出到 softmax 中，将其变为 0-1 的一个数，则变为了注意力权重。`因此，不同的评分函数会导致不同的注意力池化操作`



- 加性注意力

一般来说，当查询和键是不同长度的矢量时，可以使用加性注意力作为评分函数。

> 加性注意力使用两个 MPL 来计算 q 和 k 之间的相关性

![image-20250227152350278](https://img.leftover.cn/img-md/202502271523388.png)



- 缩放点积注意力

  使用点积可以得到计算效率更高的评分函数， 但是点积操作要求查询和键具有相同的长度 d。

  ![image-20250227152820506](https://img.leftover.cn/img-md/202502271528637.png)

`为什么除以根号d`

- 假设查询和键的所有元素都是独立的随机变量，并且都满足零均值和单位方差， 那么两个向量的点积的均值为 0，方差为 d。
  - 显然当 d 变大，q 和 k 点积的方差也会变大，这意味着向量之间元素的差值变大。
  - 元素的差值变大会导致 softmax 函数，退化为 argmax 函数。也就是最大值 softmax 后的值为 1， 其他值则为 0。使得 softmax 后的向量，成为一个[one hot 向量](https://zhida.zhihu.com/search?content_id=249933351&content_type=Article&match_order=1&q=one+hot向量&zhida_source=entity)。
  - softmax 只有一个值为 1 的元素，其他都为 0 的话，反向传播的梯度会变为 0, 也就是所谓的梯度消失

- 因此上面的梯度消失的问题是因为 q 和 k 点积的方差与 d 有关导致的，那么要解决梯度消失问题，只要让其方差变为一个常数即可，因此我们除以 $\sqrt{d}$, 让其方差变为 1。

参考资料:[为什么 Attention 计算公式中，QK 的点积要除以根号 d](https://zhuanlan.zhihu.com/p/4555299718)

# 深度学习中的专业术语

1. SOTA（state-of-the-art）：通常指在某个特定的任务上当前最优的模型和方法
2. benchmark：**指用于评估和比较多个模型性能的基准测试**。通常包含一个/多个标准数据集（例如 ImageNet 数据集），评价指标（准确率、F1 分数等）
3. baseline：指用于衡量最新模型或新方法的基准模型或基线指标。一般 baseline 是一个简单且已有的模型或方法（例如 ResNet18），作为一种最低标准，用来对比和评估新模型的大小

5. backbone： 一般指骨干网络





# 编写损失函数的一些小技巧

1. 数值差异过大（通常会取对数）

   对于房价来说，有些房子的价格很高，那么预测值和真实值误差可能会比较大，对损失函数的影响比较大，而有些房子的价格比较低，
   那么预测值和真实值误差相对来说误差没那么大，对损失函数的影响就比较小。相比之下我们更关注的是相对误差，即(y_hat - y)/y,
   因此对于这种情况，我们通常会先取对数，再放入正常的损失函数中计算损失，如下

```python
rmse = nn.MSELoss()
def log_rmse(y_hat, y):
    # 这里模型在一开始的时候可能会预测出负值，log（负数）= nan，因此要对输出进行裁剪
    y_hat = torch.clamp(y_hat, 1, float('inf'))
    y_hat = torch.log(y_hat)
    y = torch.log(y)
    l = rmse(y_hat, y)
    return l
```

# Resnet 为什么有效

1. 不用 resnet

   当我们的 net 很深的时候，通常后面的层的梯度大，他会收敛得比较快，然后前面层的梯度很小，收敛得慢。然后当后面层收敛的时候，前面层还没收敛，前面层权重的变化又会返过来影响后面的层，导致后面层的参数又白学了。因此当你的网络的层数堆叠得很深的时候，其效果可能还不如一个浅层的网络好。

2. 使用 resnet

   从下面的图可以看出，假设使用了残差块的输出为 g(x)，正常块的输出为 f(x)

   - 不使用残差块:损失对当前层的输入的导数为 $\frac{\partial l}{\partial x}  = \frac{\partial l}{\partial f(x) } \cdot \frac{\partial f(x)}{\partial x}  $ ,很明显，如果网络很深，这样会导致前面层的梯度很小

   - 使用残差块: 损失对当前层的输入的导数为 $\frac{\partial l}{\partial g(x)} \cdot \frac{\partial g(x)}{\partial x}  $

     又因为$g(x) = f(x) +x $ , 因此$\frac{\partial g(x)}{\partial x} = \frac{\partial f(x)}{\partial x}   +1 $

     因此$\frac{\partial l}{\partial x} = \frac{\partial l}{\partial f(x)}  \cdot (\frac{\partial f(x)}{\partial x}  +1 ) = \frac{\partial l}{\partial f(x)} \cdot \frac{\partial f(x)}{\partial x}   + \frac{\partial l}{\partial f(x)}  $

     这样的话就不会说导致前面层的梯度很小，导致前面层和后面层的梯度差异较大，从而导致网络收敛困难
