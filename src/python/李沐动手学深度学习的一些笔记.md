# softmax 的实现

```python
# 根据softmax的数据公式对其的实现
def softmax(X):
    X_exp = torch.exp(X)
    sum = torch.sum(X_exp, 1, keepdim=True)
    return X_exp / sum
  
  
# softmax的第二种实现
def softmax1(X):
    max_num = torch.max(X)
    X = X - max_num
    X_exp = torch.exp(X)
    sum = torch.sum(X_exp, 1, keepdim=True)
    return X_exp / sum  
```

因为指数函数的值域是（0，inf），因此可能会出现数据上溢的问题，例如 exp（50）非常大，它可能超出计算机所能表示的范围，从而被近似为无穷大（inf）。这会带来一些问题，例如在反向传播时可能会出现 NaN（不是数字）的情况。`解决这个问题的一种常用技巧是，在计算softmax之前，先从所有输入中减去输入中的最大值。这样可以确保指数函数的输入不会太大而导致数值上溢。`具体代码请看 softmax1

```python
# 交叉熵损失函数的实现
def cross_entropy(y_hat, y):
    y_hat = softmax(y_hat)
    loss = -torch.log(y_hat[range(len(y_hat)), y])
    return torch.mean(loss)
```

> 正常的交叉熵损失函数的实现其实是没有 softmax 的那一步的，`pytorch的交叉熵损失中会先对y_hat进行一个softmax的操作`，这样操作可以将输入的固定在（0，1）中，且不会出现 0，因为 ln（0）会得到-inf。因此，如果使用普通的交叉熵损失函数在实际计算时，通常忽略预测概率接近 0 的样本对损失函数的贡献，这可能会导致模型欠拟合，并且在训练过程中难以收敛。



# 初始化模型参数

```python
import torch
import torch.nn as nn

# 定义一个简单的网络
class MyNetwork(nn.Module):
    def __init__(self):
        super(MyNetwork, self).__init__()
        self.conv1 = nn.Conv2d(1, 32, 3)
        self.fc1 = nn.Linear(32 * 6 * 6, 10)

    def forward(self, x):
        x = self.conv1(x)
        x = x.view(x.size(0), -1)
        x = self.fc1(x)
        return x

# 定义初始化函数
def init_weights(m):
    if isinstance(m, nn.Conv2d):
        nn.init.xavier_uniform_(m.weight)
    elif isinstance(m, nn.Linear):
        nn.init.kaiming_normal_(m.weight)

# 创建模型
net = MyNetwork()

# 使用 apply 函数应用初始化
net.apply(init_weights)
```

def dropout_layer(X, dropout):
    assert 0 <= dropout <= 1
    if dropout == 1:
        return torch.zeros_like(X)
    if dropout == 0:
        return X
    mask = (torch.rand(X.shape) > dropout).float()
    return mask * X / (1 - dropout)

# 数值稳定性、参数初始化、激活函数的选择

1. 梯度爆炸：

   - 可能会导致值超出可表示的范围（尤其是对于 16bit float）

   - 并且梯度爆炸会导致对学习率很敏感

     <img src="https://img.leftover.cn/img-md/202502211559132.png" alt="image-20250221155916020" style="zoom:50%;" />

2. 梯度消失：

- 梯度值变为很小，趋近于 0，尤其对 16 位浮点数
- 因为梯度很小，导致不管如何选择学习率，参数都不怎么更新
- 对于前面层尤为严重，因为可能后面层的梯度没那么小，后面层的训练效果还是比较好，而对于前面层梯度就很小，导致前面层的参数基本不更新。这也就导致了无法让神经网络变得更深





解决方法：我们可以让每一层的输出和每一层的梯度保持为一个均值为 0，方差为一个固定值。这样就可以有效地控制数值的范围

经过一系列推导之后要达到上述的要求，我们要保证 $n_{t-1}\gamma_{t} =1 $ 和 $n_t\gamma_{t}=1 $ ,即要保证每一层的输入和输出的维度一致，显然这是不可能的。

`Xavier`则使用了一种折中的方法，从而让每一层的权重参数的方差满足$\frac{n_{t-1}+n_t}{2}$ (（即输入维度+输出维度）/2)

# nan ,inf 的产生及解决

inf 可能是学习率太大了，或者权重初始化的时候权重太大了

nan 一般是除以了 0 导致的，通常是 0/0 导致的 或者 log（负数）也会变成 nan



**解决方法：**

- 合理地初始化参数

- 不断调小学习率
