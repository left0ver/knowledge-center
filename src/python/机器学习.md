# 线性回归

## 梯度下降

梯度下降通过迭代地调整模型参数，沿着损失函数的负梯度方向移动，以找到损失函数的**全局最小值**或**局部最小值**（因为梯度的负方向是下降最快的）

梯度下降分为批量梯度下降、随机梯度下降、小批量梯度下降

<img src="https://img.leftover.cn/img-md/202410061520858.png" alt="image-20241006152030734" style="zoom:50%;" />

- 批量梯度下降：损失函数的方向相对稳定，但是计算量很大，每次都要计算所有的数据量

- 随机梯度下降：每次随机找一个样本，迭代的速度很快，但不一定每次都朝着收敛的方向

- 小批量梯度下降：每次找部分的数据进行计算，综合了批量梯度下降和随机梯度下降的优点，计算效率也不错，损失函数也相对稳定

  **我们需要设置每次迭代需要计算的样本数量，常用的有 64，128，256，在能容忍的范围内越大越好，越大则训练出来的模型越精确**

> 在梯度下降中学习率很重要，学习率决定了每次更新的步长
>
>如果学习率 **太小**，梯度下降的步长就很小，导致收敛速度很慢，需要很多迭代才能找到最优解。
>
>如果学习率 **太大**，每次更新的步长可能过大，导致跳过了最优解，甚至可能不收敛，出现震荡。

> sk-learn 中可以设置动态学习率，learning_rate =`invscaling` 表示逐渐递减的学习率，学习率会随着迭代次数的增加而减少
>
>learning_rate =`adaptive` 即当损失函数在一个迭代周期内没有降低时，学习率会缩小（即动态适应的学习率）

⚠️学习率应该尽量地小，随着迭代的进行应当越来越小





## 正则化来减小过拟合风险

正则化通过在损失函数中添加对模型复杂度的惩罚项，用于限制模型的参数大小，从而降低模型复杂度，使其更加简化且不容易过拟合

#### 常见的正则化方法

1. L1 正则化（Lasso 正则化）

   <img src="https://img.leftover.cn/img-md/202410070854208.png" alt="image-20241007085450048" style="zoom:50%;" />

   **效果：** Lasso 正则化会将一些参数缩小为 0，从而使得模型变得更加稀疏。因此，它非常适合用于特征选择（自动选择对模型影响较小的特征，使它们的权重变为 0）。

   > 常用于线性回归、逻辑回归

   

2. L2 正则化（Ridge 正则化）

   <img src="https://img.leftover.cn/img-md/202410070855508.png" alt="image-20241007085507458" style="zoom:50%;" />

   **效果：**Ridge 正则化将模型参数缩小到非常小的值，但不完全等于 0，从而抑制模型的复杂度和防止过拟合，Ridge 正则化在处理高度相关的特征时表现较好

   > 常用于线性回归、逻辑回归、神经网络

3. Elastic Net 正则化

   Elastic Net 正则化是 L1 正则化和 L2 正则化的结合

   <img src="https://img.leftover.cn/img-md/202410070906580.png" alt="image-20241007090613510" style="zoom:50%;" />

   **效果：**Elastic Net 同时具有 L1 和 L2 正则化的优点，即可以实现特征选择（稀疏性）和防止过拟合。

   > 虽然 Elastic Net 正则化同时具有 L1 和 L2 正则化的优点，但是 Elastic Net 需要调整 2 个参数：λ1 和 λ2

   > 常用于线性回归

# 决策树

<img src="https://img.leftover.cn/img-md/202410092315660.png" alt="image-20241009231508596" style="zoom:33%;" />

决策树是一个比较万能的模型，它即可以做分类，也可以做回归（主要是做分类）

**熵：**熵是表示随机变量不确定性的度量（即不确定性越大，熵越大）

计算熵的公式

<img src="https://img.leftover.cn/img-md/202410092308180.png" alt="image-20241009230847067" style="zoom:33%;" />



## 如何构建决策树

1. 根据什么选择对应的节点？

   根据信息增益，计算出剩下的所有特征的信息增益，**选择信息增益最大的那个特征**即可

> **信息增益：**当前节点的熵 - 分裂之后所有子节点的熵的和



## 遇到连续值如何选取分界点

1. 首先将连续的值进行升序排序

2. 确认候选的分界点，通常选择**相邻两个值的均值**作为候选分界点

   <img src="https://img.leftover.cn/img-md/202410092329955.png" alt="image-20241009232906881" style="zoom:50%;" />

3. 对每个分界点，将数据集划分为 2 部分，**计算划分后的信息增益或 gini 系数，选择信息增益最大/gini 系数最小的分界点进行划分**

## 决策树是如何解决回归问题的？

- 和处理连续值的时候一样，将连续的值进行生序排序
- 确认候选的分界点，通常选择**相邻两个值的均值**作为候选分界点
- **对每个候选分界点计算它分裂之后的子节点的方差，找到 MSE 最小的分界点**，对每个特征都进行这样的计算，最后找到最优的切分变量和切分点
- 如何进行预测？
  - 通过传入的数据的特征与决策树的节点进行对比，最后找到一个叶子节点，预测值就是叶子节点的样本中所有 label 的平均值

## 决策树的剪枝策略

**为什么要剪枝？**

决策树的过拟合风险很大，理论上若树足够庞大，每一个叶子节点一个数据，这样在训练集上的正确率很高，但是容易过拟合。因此我们需要用剪枝操作让决策树不要那么庞大，从而减低过拟合的风险

剪枝策略分为：预剪枝、后剪枝

预剪枝：变建立决策树边进行剪枝操作（常用）

- 常用的预剪枝操作有：限制决策树的深度，限制叶子节点的个数、限制叶子节点的样本数、信息增益量等

  > 例如信息增益< 某个值就不再继续向下分裂了

后剪枝：建立完决策树之后进行剪枝操作

一般是根据一些衡量标准，去掉一些叶子节点

<img src="https://img.leftover.cn/img-md/202410092322110.png" alt="image-20241009232249033" style="zoom:33%;" />



## 决策树算法的选择

**ID3 算法：信息增益 **的问题：

- 信息增益在进行特征划分时，容易偏向于取值更多的特征，即“高基数的特征”，例如某个特征有大量的唯一取值，划分之后的子集非常纯，熵会显著降低，从而导致较高的信息增益。这种情况下信息增益往往会选择这种高基数的特征，即使这些特征对预测并没有太大的帮助



**C4.5 算法：信息增益率**

- 信息增益率 = 信息增益 / 特征取值的熵，像上述这种“高基数”特征的情况，会有很高的信息增益，信息增益率 = 信息增益 / 原来特征取值的熵

  > 例如。特征 A 有 3 个取值：低、中、高，分为 3 类
  >
  >原来特征的熵就是 A 的熵，如果原来的特征只有少量取值，那么它的熵就回很低，否则熵就很高，因此信息增益率就越低

  **新的问题：**信息增益率对属性取值较少的时候会有偏好，

     为了解决这个问题，C4.5 不是直接选择信息增益率最大的属性进行划分，而是先通过信息增益筛选一遍，先把信息增益低于平均值的剔除掉，之后从剩下的属性中选择信息增益率最高的属性进行划分

<img src="https://img.leftover.cn/img-md/202410092333807.png" alt="image-20241009233329752" style="zoom:67%;" />



## 决策树中的正则化（预剪枝）

- **min_samples_split**：默认 2 （节点在分割之前必须具有的最小样本数）
- **min_samples_leaf**： 默认 1 （叶子节点必须具有的最小样本数）
- **max_leaf_nodes**： 默认 None （叶子节点的最大数量）
- **max_depth**： 默认 None （决策树的最大深度）
- **max_features**: 默认 None (寻找最佳切分点时考虑的特征的数量) ***一般这个值不会设置，除非特征非常多*

# 支持向量机（SVM）

### 什么是支持向量？

支持向量是距离超平面最近的那些数据点

对于上面的这个图，很明显最上面的那个圆圈是异常点，因此虚线的决策边界会比实线的决策边界更好，正如上面所说，svm 是要找的一个超平面，将数据点完全区分，这种方法有点过于严格了，因此我们引入**松弛因子**来改善这种情况

$\xi _i$ 就是松弛因子。

- 当 C 很大的的时候，则表示松弛因子需要比较小，当松弛因子为 0 时，即相当于没有引入松弛因子，则表示要严格分类
- 当 C 很小时，则表示松弛因子需要比较大，此时就意味着有更大的错误容忍



## SVM 调参

- *kernel* 使用`rbf` （默认的），高斯核函数

- C：默认 1.0 ，可以根据实际情况调整（C 越大，则意味着分类严格，不能有错误，容易导致过拟合； C 非常小时，意味着可以有很大的错误容忍，容易导致欠拟合）

- *gamma*：默认为`scale`,即根据样本数量和样本方差进行计算得出，也可以手动指定一个值，不宜过大

  > gamma 越大，则模型的决策边界会更加弯曲，可能会导致过拟合
  >
  >gamma 越小，模型的决策边界会更加平滑，可能会导致欠拟合



# 集成算法

## Bagging（Bootstrap Aggregating）

典型代表就是随机森林，简单来说就是平行训练多个模型，然后对多个模型的预测的结果取平均（回归） 或 少数服从多数（分类）

> 训练多个模型不是说每个模型都一样，**而是随机选取训练数据，随机选取特征。**   例如训练集总共有 1w 数据，15 个特征；则每个模型随机选取里面的 6k 个数据，10 个特征进行 fit，最后的结果取总的平均值

- 随机森林

  思想跟上面的差不多，随机森林使用的是很多个决策树，将很多棵树并行放在一起

  <img src="https://img.leftover.cn/img-md/202410110903410.png" alt="image-20241011090301278" style="zoom: 50%;" />

- bagging 算法和决策树的对比

## Stacking

Stacking 将多个不同种类的基学习器组合起来，最终由一个元学习器（**通常是线性回归/逻辑回归等比较简单的模型**）根据基学习器的输出进行预测。**每个基学习器的输出作为元学习器的输入**

> 元学习器将每个基学习器的输出作为一个特征，进行学习，从而输出最终的结果

优点：可以综合多个不同的模型，提升性能，一般用于竞赛

缺点：计算复杂度高，训练慢

```python
from sklearn.datasets import make_moons, fetch_openml
from sklearn.ensemble import VotingClassifier, RandomForestClassifier, BaggingClassifier, StackingClassifier
import matplotlib.pyplot as plt
from sklearn.linear_model import LogisticRegression
from sklearn.tree import DecisionTreeClassifier
from sklearn.metrics import accuracy_score, classification_report
from sklearn.svm import SVC
import numpy as np

mnist = fetch_openml("mnist_784", as_frame=False)

X = mnist.data
y = mnist.target

X_train, X_test, y_train, y_test = X[:6000], X[6000:], y[:6000], y[6000:]
print(y_train.shape)
model = LogisticRegression(max_iter=70000)
# 构建stacking模型
model = StackingClassifier(
  [("ran_for", RandomForestClassifier(300)), ("svc", SVC()), ("dec", DecisionTreeClassifier(max_depth=6))],
  final_estimator=LogisticRegression(max_iter=5000))

model.fit(X_train, y_train)


y_pred = model.predict(X_test)
print(classification_report(y_test, y_pred))
```

> ⚠️通过对识别数字这个数据集的比较，stacking 算法的准确率最高，可达 95%，综合评分也是 95%

# 模型的评估

## 交叉验证

#### 什么是交叉验证？

我们在可能有多个模型可以选择，我们需要评估各个模型的好坏（或者评价某个模型的好坏），因此我们会从训练集中拿出一部分数据（例如是 20%）当作验证集（train-dev），然后我们会在训练集上训练，在验证集上进行预测。同时为了能够更加准确地对模型进行评估，我们往往会进行多次验证，如下图所示：取训练集的不同的部分作为验证集，剩余的作为训练集，进行多次训练，取整体数据的平均值（例如取正确率的平均值，均方根误差的平均值），以此来评价这个模型的好坏。

> 一方面，验证集的数量不能太小，太小的话不能准确地反映出模型的好坏，太大的话会导致训练的数据很小，从而导致模型整体效果很差。
>
>另一方面，验证的次数也不能太多，太多则会费时间，太少的话可能对模型的评估不准确

<img src="https://img.leftover.cn/img-md/202410061450649.png" alt="image-20241006145055505" style="zoom:50%;" />

####  K 折交叉验证（K-Fold Cross-Validation）

- **K 折交叉验证** 是最常用的交叉验证方法。
- 将数据集划分为 **K** 个等大小的子集（折），其中 **K-1** 个用于训练，剩下的一个用于验证。
- 重复 **K** 次，每次选择一个不同的子集作为验证集，其余的用于训练。
- 最后，将 **K** 次的验证结果取平均，作为模型的最终评估结果。

#### 交叉验证的优缺点

##### 优点

**评估稳定性**：通过多次训练和验证，交叉验证可以使模型评估更加稳健，减少因为训练集和测试集划分不当带来的偏差。

**减少过拟合风险**：由于交叉验证让模型在不同的训练集和验证集上训练和评估，可以有效减少过拟合的风险。

##### 缺点

- 比较费时间
- 计算成本大

#### ROC 曲线

ROC 曲线是**二分类**中常用的评估方法，y 轴为 TPR ，x 轴为 FPR

TPR = TP/（TP+FN） 即 recall

FPR = FP/（FP+TN）

要绘制 ROC 曲线，我们要先使用 roc_curve()函数计算出各种阈值的 TPR 和 FPR

```python
import numpy as np
import matplotlib.pyplot as plt
from sklearn.datasets import make_classification
from sklearn.model_selection import train_test_split
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import roc_curve, auc

# 生成二分类数据集
X, y = make_classification(n_samples=1000, n_features=20, random_state=42)
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

# 创建并训练逻辑回归模型
model = LogisticRegression()
model.fit(X_train, y_train)

# 获取预测概率
y_scores = model.predict_proba(X_test)[:, 1]

# 计算 ROC 曲线和 AUC
fpr, tpr, thresholds = roc_curve(y_test, y_scores)
print(thresholds.shape)
print(y_test.shape)
# 计算AUC
roc_auc = auc(fpr, tpr)

# 计算 Youden's J statistic （通常选择TPR与FPR差值最大的阈值）
# 找到对应的最佳阈值
youden_j = tpr - fpr
best_threshold_index = np.argmax(youden_j)
best_threshold = thresholds[best_threshold_index]
print(f"最佳阈值: {best_threshold}")

# 绘制 ROC 曲线
plt.figure(figsize=(10, 6))
plt.plot(fpr, tpr, color='blue', lw=2, label=f'ROC curve (area = {roc_auc:.2f})')
plt.plot([0, 1], [0, 1], color='grey', linestyle='--')  # 参考线：随机分类的表现
plt.xlim([0.0, 1.0])
plt.ylim([0.0, 1.05])
plt.xlabel('False Positive Rate')
plt.ylabel('True Positive Rate')
plt.title('Receiver Operating Characteristic (ROC) Curve')
plt.legend(loc='lower right')
plt.grid()
plt.show()
```

> 我们通常选择 TPR 与 FPR 差值最大的阈值。
>
>同时根据不同的业务需求，可能需要在**精确率（Precision）** 和 **召回率（Recall）** 之间进行权衡。在这种情况下，可以手动选择适合的阈值，例如：
>
>- 如果**错过正类的代价高**（如医疗诊断中的漏诊），可以选择具有较高 TPR 的点，即使 FPR 较高。
>- 如果**误报的代价高**（如垃圾邮件过滤），则选择较低 FPR 的点。
>
>在绘制的 ROC 曲线上找到符合要求的点，并从 `thresholds` 数组中找出对应的阈值即可。



#### ROC 的 AUC

AUC 是 ROC 曲线围成的面积，一个好的分类器的 AUC 应该尽量=1，纯随机的分类器的 AUC 等于 0.5

### 对分类数据进行编码

如果数据之间有大小的关系，例如成绩分为高、中、低 ，我们可以将其编码为 3 ,2, 1 。 若数据之间没有大小的关联，则不推荐这样编码。这样子的话可以使用独热编码

### 将连续变量离散化

有些算法只能处理离散型的数据，例如决策树、朴素贝叶斯等，因此我们需要将连续的数据进行分段，使其变为一段段离散型的区间

1. 等宽法
   - 例如 20-30 岁的年龄为一类，30-40 岁的年龄为一类（分布比较均匀的数据）
2. 等频法
   - 将连续变量分为若干个区间，每个区间包含的样本数大致相同。例如将数据分为包含相同样本数的 4 个区间（可以较好地处理数据不均匀的问题，确保每个区间都有足够的样本数）
