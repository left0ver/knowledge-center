# 评价指标

## IOU

IOU 用来评价预测的框与真实的标签的框的重叠程度，重叠程度越高，则预测越准确，IOU = 重叠的面积/（预测框与真实框并起来的面积）

<img src="https://img.leftover.cn/img-md/202411231020186.png" alt="image-20241123102022042" style="zoom:33%;" />

## 精度

衡量检测的准确度

## Recall

它衡量模型在所有实际存在的目标中，能够被正确检测到的比例。召回率反映了模型漏检的情况，即有多少真实目标未被检测到。

## MAP（mean Average Precision）

通过调整模型的置信度阈值，可以得到不同的精确率和召回率值，从而绘制出精确率-召回率曲线。曲线下的面积即为**平均精度（Average Precision, AP）**。

MAP 是对多个类别的 AP 求平均得出的指标，用于衡量模型在所有类别上的整体性能。

## 知识蒸馏

我们现实中存在一些精确度很高的大模型，这些模型通常精确度很高，但是速度很慢，不适合应用，**而知识蒸馏就是通过大模型来协助小模型的训练，提高小模型的精度**

如下图所示：

1. 训练过程中将图片输入，通过教师模型进行预测，得到 soft labels，同时也通过学生模型（小模型）进行前向传播，得到 soft preictions，将 soft labels 和 soft preictions 求一个损失得到 L1，

2. 同时图片通过小模型前向传播得到 hard prediction ，再与 hard labels（正常的标签数据）求损失，得到损失 L2

   > 这个就是正常的训练过程

3. 将 L1 和 L2 通过一定的比例进行相加，得到最后的 Loss，最后再进行反向传播

> 这里求 soft label 和 soft prediction 的方法和求 hard prediction 的方法有些许不同，会通过一些变化，使得 soft labels 和 soft prediction 的结果更加平缓，相差不会太大

<img src="https://img.leftover.cn/img-md/202411281344639.png" alt="image-20241128134405555" style="zoom: 40%;" />



**知识蒸馏的应用：**

1. 可以用于模型压缩

2. 优化训练，防止过拟合

3. 加快小模型的收敛速度

4. 海量数据无样本学习

   > 这里为什么能用于海量样本无样本学习呢？
   >
   >根据上面的图可以看出，小模型输出的 soft prediction 和大模型输出的 soft labels 可以求得一个损失，那么我们是不是可以只用这一个损失来进行反向传播，相当于我们用一个大的模型来训练一个小模型

## 模型重参数化

在训练阶段，一些模块（RepVGG）可能采用多分支结构以增强特征融合效果；在推理阶段，通过重参数化技术将多分支结构转换为单一分支，从而简化计算过程，提高推理速度。

## L1 距离和 L2 距离

对于两个 n 维向量 $\mathbf{A} = (x_{1}, x_{2}, \ldots, x_{n})$ 和$ \mathbf{B} = (y_{1}, y_{2}, \ldots, y_{n})$ 

### L1 距离

对异常值不那么敏感，适用于存在离群点的数据
$$
L1 = \sum_{i=1} ^ n  | x_i -y_i|
$$

### L2 距离

也叫欧几里得距离，对异常者敏感
$$
L2 = \sqrt {\sum_{i=1} ^n (x_i-y_i)^2}
$$

## anchor free

### 基于关键点的 Anchor free 检测算法

1. 代表作：[Corner Net](https://arxiv.org/abs/1808.01244)

- 图片会经过 conv 输出两组 heapmap（分别对应左上角和右下角的 heatmap），不同于 feature map，Heatmaps 预测了每个位置属于哪一类的概率。假如 Heatmaps 的维度为 W×H×C，C 为通道数，也就是类别数
- 之后通过 embeddings 模块，为每个检测到的 corner 预测一个嵌入向量，若两个 corner 属于同一个物体，那么这两个 corner 的嵌入向量的距离很小

- max pooling 和 avg pooling 用于增大感受野，缩小特征图的尺寸，但是角点不一样，要确定一个像素是否是 top-left corner，我们需要向右寻找物体的上边界，向下训练左边界，即每个像素向右扫描最大池化（找到右边像素的最大值），向下扫描最大池化（找到下面像素的最大值），这该像素点经过 corner pooling 之后的值为这两个值相加。

### 基于中心点的 Anchor Free 算法

1. [FCOS](https://arxiv.org/abs/1904.01355)

   ![image-20241128230800860](https://img.leftover.cn/img-md/202411282308010.png)

   **标签分类：**FCOS 把每一个位置都当作训练样本，如果某个位置（x，y）落在任何 ground-truth 内，则视为正样本，否则视为负样本（背景）。**FCOS 对每个位置到 ground-truth 的四条边的距离作为回归目标。**

   > 倘若一个位置落在多个 ground-truth 内，则视为模糊样本，那么这时候会简单地选择面积最小的 ground-truth 作为其回归的目标

   **FPN（特征金字塔）的网络结构减少重叠的歧义的产生**：例如图片中有一个老鼠、一个大象，有某个点落在了这两个 Ground-truth 上面，而如果我们使用了 FPN 之后，我们就可以将大象放在一个深度的特征图上预测，老鼠放在一个浅的特征图上预测。使用 FPN 多级预测大大减少了上述的歧义

   > 当然也可能重叠的两个物体的大小差不多，两个物体使用的是同一个检测头预测，那么这时候我们就选择面积最小的 ground-truth 作为其回归的目标

   **centerness 抑制低质量的目标框**

   - 在 FCOS 中，会将 ground-truth 中的点平等地视为正样本，这样就会导致产生很多离目标框中心距离较远点低质量目标框。FCOS 中提出 center-ness 的策略来抑制这些低质量的目标框。

   centerness 描述了当前像素点到其负责的 ground-truth 对象的中心点归一化距离，为 1 则表示距离很近（重合），为 0 则表示距离很远。使用 BCE 损失进行训练。
   $$
   centerness^* = \sqrt { \frac{min(l^*,r^*)}{max(l^*,r^*)} × \frac{min(t^*,b^*)}{max(t^*,b^*)} }
   $$

   - $l^* $ 是到 ground-truth 左边框的距离，$r^*$ 是到 ground-truth 右边框的距离，$t^*$ 是到 ground-truth 上边框的距离，$b^*$ 是到 ground-truth 下边框的距离，

    

   - **推理时：**最终得分（用于对检测到的边界框进行排序）是通过将预测的中心度与相应的分类得分相乘来计算的。因此，中心度可以降低远离物体中心的边界框的得分。因此，这些低质量的边界框很可能在最终的 NMS（非极大值抑制）过程中被过滤掉，从而显著提高检测性能。

   

   **总结：**核心思路就是使用检测点到真实框四边的距离进行回归

   分类+回归分支：

   - ground-truth 中的所有的点都平等地视为正样本，其他点视为负样本。
   - 使用 FPN 进行多尺度预测，从而减少重叠的歧义的产生

   中心度分支：提出 centerness 的概念，最后得分依赖于 centerness，centerness 分数低的预测框容易在推理阶段使用的 NMS 过滤掉

   

2. [Center Net](http://arxiv.org/abs/1904.07850)

   主要的流程：

   - 输入图像生成特征图，再基于特征图生成热力图，表示每个类别中心点的概率分布`（每个像素的值表示该位置为目标中心点点概率，目标类别由热力图的channel来决定）`

   - 提取热力图的局部最大的点（即目标的中心点），这样就可以知道这个区域使用哪个中心点产生预测框，因此就不需要 NMS 这个费时间的后处理过程

     > 在 center net 中只把中心点当作正样本，而 corner net 中除了中心点之外，还会通过圆形的高斯分布来生成一个监督信息（权重），越靠近中心点，这个监督信息就越接近 1

   - 对于检测到的中心点，**通过回归预测 Bounding Box 的宽高。**并使用 L1 损失函数计算损失

   - 损失函数：

     - heatmap loss：采用 Focal loss
     - 边界框的尺寸损失：L1 损失
     - offset Loss：L1 损失

## ATSS（Adaptive training sample selection）（动态样本分配）

[ATSS](http://arxiv.org/abs/1912.02424) 通过消融实验证明了 anchor based 和  anchor free 其实无所谓，**导致性能差异的原因是正负样本的分配**，并且证明了在图像上的每个位置放多 anchor 是一种无用的操作，并且提出了一种自适应的样本选择的方法

### 算法过程

1. 对于图像上的每个 ground-truth，先找到其候选正样本: 即在每一层金字塔中，根据 L2 距离选择中心最接近 ground-truth 的`k` (k 是一个超参数) 个 anchor box，假设有 m 层金字塔，则总共选择 k×m 个候选正样本。

2. 计算这些正样本和 ground-truth 的 IOU，并且求得这些 IOU 的均值 $\mu $ 和标准差 $\sigma $,计算阈值 $t_g = \mu  +\sigma $ ,再从原本划分的候选正样本选择 IOU > $t_g$ 的 anchor 作为正样本，剩下的全部为负样本。

   > 若一个 anchor 被划分给了多个 ground-truth，则选择 IOU 最高的一个

   <img src="https://img.leftover.cn/img-md/202411291634482.png" alt="image-20241129163455453" style="zoom:67%;" />



### 这样设计的原因

1. 如果一群 anchor box 有比较高的 $\mu $ ,说明这些 anchor 的质量比较高，因此 IOU 的阈值应该比较高，而若 $\mu $ 比较低，则表明大多数的候选正样本的质量比较低，因此 IOU 的阈值也应该比较低。
2. $\sigma $ 可以用来衡量哪些层适合检测该对象的度量，高的$\sigma $ 意味着方差比较大，有一个金字塔的 IOU 比较高，其他大多数的金字塔的 IOU 比较低，意味着存在一个专门的金字塔的 anchor 来检测。$\sigma $ 比较低，则表示每层的候选正样本的 IOU 相差不大，意外着存在好几个适合检测该对象的金字塔。

> 因此使用 $\mu +\sigma $  可以自适应低在适合的金字塔上选择正样本

## simOTA（动态标签分配）

1. 先确定正样本的候选区域（使用中心先验），即选择每个 ground-truth 的中心点，以这个中心点为准，作一个 5*5 的正方形称为 fixed center area，因此 ground-truth + fixed center area 所有的网格都为正样本的候选区域

2. 求得 `每个网格的预测结果与对应的ground-truth的IOU` 和 `每个网格对每个GT的Reg+Cls loss` 从而计算 cost，cost 计算公式如下: （论文中的 $\lambda$ =3）
   $$
   c_{ij} = L_{ij}^{cls} + \lambda L_{ij}^{reg}
   $$

3. 取前 10 个，将这 10 个 IOU 求和再取整，得到当前 Ground-truth 的 `dynamic_k` ，最小为 1 (保证一个 GT 最少有 1 个正样本)

4. 为每个 GT  取 cost 最小的前 `dynamic_k`  个样本作为正样本，其他的均为负样本。

> 1. simOTA 可以做到自动地分析每个 gt 要拥有多少个正样本，若多个候选样本的 IOU 都比较高，则`dynamic_k`比较高，取的正样本就多
>2. 引入了中心先验，一般位于 GT 中心的候选区域的质量比较高



## TODO 标签分配策略（软标签分配方式）



### 以往的标签分配策略的有两个局限性

1. 分类和定位一般是用两个独立的分支，这使得两个任务缺乏交互性，预测的时候就会出现不一致性。分类得分高的预测位置不准，而预测位置准的分类得分不高
2. 在划分正负样本的时候，anchor-free 一般通过 anchor point 到目标的中心距离来划分样本，而 anchor-based 方法一般通过 IOU 来划分样本，但是最佳分类和最佳定位的 anchor 往往不一致，并且最佳的 anchor 可能并不在目标中心。因此在 NMS 阶段，精度较高的 bbox 可能会被精度低的 bbox 抑制。



### 任务对齐的度量指标

提出了一种对齐度量指标，用来衡量 anchor 级别的任务对齐程度。并且该度量标准会被集成到正负样本分配和 loss func 中，从而动态优化每个 anchor 的预测.
$$
t = s^\alpha  × u ^ \beta
$$

- $s $ 和 $u$ 分布表示分类得分和 IOU 值。$α$ 和 $β$ 用于控制两个任务在锚点对齐度量中的影响。

  > 它鼓励网络从联合优化的角度动态关注高质量（即高分类分数和高 IOU）（即任务对齐）的锚点。



### 标签分配策略

**对于每个 ground-truth，选择具有最大 t 值的 m 个 anchor 作为正样本，而将剩余的 anchor 用作负样本**



### 损失函数

为了提高对齐 anchor 的分类分数，同时降低未对齐 anchor（即 t 值较小）的分类分数，在训练过程中，我们将正样本 anchor 的二值标签替换为 t 值。但是，当正样本 anchor 的标签（即 t 值）随着α和β的增加而变小时，网络无法收敛。因此，我们使用归一化后的 t，即 t̂，作为正样本 anchor 的标签。归一化的方法：t̂的最大值等于每个实例中最大的 IoU 值（即 u）。

- 正样本的分类损失函数

$$
L_{cls\_pos}  = \sum _{i=1} ^ {N_{pos}}  BCE (s_i,\hat{t_i})
$$

- 同时为了缓解正负样本的不平衡，还采用了 focal loss，最终分类任务的损失函数如下：
  $$
  L_{cls} = \sum_{i=1}^{N_{pos}} |\hat{t_i} - s_i |^\gamma  BCE(s_i,\hat{t_i}) +  \sum_{j=1} ^ {N_{neg} } s_j^\gamma BCE(s_j,0)
  $$
  



- 回归任务的损失函数：

  一个由良好对齐的锚点（即具有大的 t ）预测的边界框通常具有大的分类得分和精确的位置，并且这样的边界框在 NMS 过程中更有可能被保留。因此，我们通过关注良好对齐的锚点（具有大的 t ）来提高任务对齐和回归精度，同时减少在边界框回归过程中不良对齐锚点（具有小的 t ）的影响。
  $$
  L_{reg} = \sum _{i=1} ^ {N_{pos}} \hat{t_i} L_{GIoU} (b,b_i)
  $$

   > 我的理解：当 t 比较大时，说明对齐效果比较好，若此时 GIoU 比较大，则损失很大，这样会让模型*更关注那些分类和定位对齐度高但定位误差大的 anchor*
   >
   >而对于 t 值比较小的 anchor，即使此时 GIoU 比较大（定位误差较大），相乘之后就会损失比较小，模型也不会过度关注

- 总的损失：
  $$
  L=L_{cls} + L_{reg}
  $$

# 目标检测任务中的损失函数

## 分类损失

### 二元交叉熵（BCE）

$$
BCE(p,y) =\left\{\begin{matrix} 
 	-log(p), \space if \space  y = 1 \\  
  -log(1-p), \space  otherwise
\end{matrix}\right.
$$

- p 是模型对 y=1 的类别的预测概率



但是对于交叉熵来说，我们的数据中可能存在类别的不平衡，例如负类 >> 正类，因此负样本就会在损失函数中占据主导地位，模型在训练的过程中就会倾向于样本多的类别，造成模型对少样本类别的性能较差



### 平衡交叉熵（Balanced cross entropy）

在 CE 的基础上增加一个 $\alpha$ 参数，用于平衡正负样本的比例
$$
L=\left\{\begin{matrix} 
 	-\alpha \cdot  log(p), \space if \space  y = 1 \\  
  -(1-\alpha) \cdot log(1-p), \space  otherwise
\end{matrix}\right.
$$
$\alpha$ 的取值，在本例子中
$$
\alpha = \frac{N_{neg}}{N_{pos} + N_{neg}}
$$

> 若负样本比较多，正样本比较少，则 $\alpha$ 比较大，则正样本在损失函数中的权重比较大



### Focal Loss

focal loss 也是为了解决样本不均衡问题，但是其思路和 balanced cross entropy 不太一样. 在
$$
L_{fl} = \left\{\begin{matrix} 
 	-(1-p)^\gamma \cdot log(p), \space if \space  y = 1 \\  
  - p^\gamma \cdot log(1-p), \space  otherwise
\end{matrix}\right.
$$
令
$$
p_t = \left\{ \begin{matrix} 

  p , \space if \space y =1 \\
	1-p, \space  otherwise

\end{matrix}\right.
$$
则 focal loss 统一为
$$
L_{fl} = -(1-p_t)^\gamma \cdot \log(p_t)
$$
$\gamma$ 为可调节因子，一般取 2。

> 在目标检测中，大多数是比较容易区别的背景，
>
>- 当$p_t$ 比较大，说明比较容易区分，损失相比于 CE 小的多（小 100-1000 倍），因此这会反过来让模型更加关注误分类的例子
>- 当$p_t$ 约为 0.5 时，损失是 CE loss 的 0.25 倍，因此模型就会更加关注这种难以区分的样本。
>- focal loss 相当于增加了难分样本在损失函数的权重，使得损失函数倾向于难分的样本，有助于提高难分样本的准确度



<img src="https://img.leftover.cn/img-md/202412151449654.png" alt="image-20241215144910509" style="zoom:33%;" />



在具体的实验中，使用了 focal loss 的一种变体，加上了一个平衡因子 $\alpha$ 用于平衡正负样本的损失贡献,一般设为 0.25。

- 正样本的权重变为 0.25。虽然看起来较小，但由于正样本数量少，这个权重**提升了每个正样本对损失函数的贡献**。
- 负样本的权重为 1 - $\alpha$ = 0.75 ，虽然数值更大，但由于负样本数量庞大，单个负样本的贡献被稀释了。



##  回归损失

### MAE （L1）

$$
L = \frac{1}{n} \cdot \sum_{i=1}^{n} |y_i -\hat{y_i} |
$$

> 1. L1 Loss 对离群点不敏感，不会放大离群点点影响
>2. 由于梯度固定，无法区分大误差和小误差，导致参数更新幅度较小，优化速度较慢。

### MSE (L2)

$$
L = \frac{1}{n} \cdot \sum_{i=1}^{n} (y_i -\hat{y_i} )^2
$$

> 1. 其损失的梯度和误差成正比，大误差的梯度大，这样可以加快收敛
>2. L2 损失会对误差做平方项处理，大的误差会被放大，对离群值会更敏感。

### Smooth L1 Loss

$$
L = \left \{\begin{matrix}
0.5x^2,\quad |x| <1 \\
|x| -0.5, \quad |x| >=1

\end{matrix}\right.
$$



- $x = y - \hat{y}$ ，表示真实值与预测值之间的误差。

> Smooth L1 Loss 结合了 L1 和 L2 的优点，在误差较小时使用 L2 损失，在误差较大时使用 L1 损失。
>
>1. 在误差较大时使用 L1 损失，减少了离群点的影响
>2. 小误差时使用 L2 损失，梯度平滑，收敛更快





### Balanced L1 Loss


$$
L_b(x) = \left \{\begin{matrix} 

\frac{\alpha}{b} \cdot (b|x| + 1) \ln (b|x|+1) - \alpha|x| , \quad if \space |x| <1 \\
\gamma |x| +C ,\quad otherwise

\end{matrix}\right.
$$

> 总体损失 = 回归 + 分类的损失。将损失值>=1 的样本称为异常点，其他的样本称为内点。
>
>一种平衡这两种任务的方法就是调整他们的损失权重，但是由于回归任务的无边界性，若直接提高定位损失的权重会让模型对异常值更加地敏感。这些异常值，可以被视为困难样本，会产生过大的梯度，这对训练过程有害。内点，可以被视为简单样本，与异常值相比，对整体梯度的贡献很小。更具体地说，内点平均只贡献每个样本 30%的梯度（与异常值相比）

Balanced L1 Loss  的思想是提高内点（即准确样本）的梯度，从而平衡分类和回归任务。其梯度计算如下：

![image-20241215172301481](https://img.leftover.cn/img-md/202412151723577.png)

- 在$\alpha$ 的控制下，可以增加内点的梯度，并且不影响异常点的梯度。
- $\gamma $ 用于控制回归误差的上界
- b 用来保证上面 $L_b$ 损失的两个公式在(x=1)的时候有相同的值

> 论文中的默认参数为: $\alpha =0.5$ 、 $\gamma =1.5$ 



### IoU Loss

IOU: 交并比。反映了两个框的重叠程度
$$
L = 1 -IOU
$$



### GIoU Loss

IOU 存在的问题：

- IOU 并不能反映两个框重叠时候的形状是否彼此相近或者相距甚远,如下图所示：
- 如果两个框没有相交，则 IoU =0，$L_{IoU} = 1 -0 =1 $，不能反映两者的重合度



<img src="https://img.leftover.cn/img-md/202412151602114.png" alt="image-20241215160201013" style="zoom:50%;" />

$$
GIoU = IoU - \frac{C- (A\bigcup B ) }{C}
$$

$$
L_{GIoU} = 1- GIoU
$$

- C： 同时包含了预测框和真实框的最小框的面积
- $A\bigcup B$ : 两个框并集的面积

<img src="https://img.leftover.cn/img-md/202412151606296.png" alt="image-20241215160606203" style="zoom: 33%;" />

> 1. 解决了原始 IoU 重合面积相同，但是姿态不同的问题

### DIoU Loss

IoU 和 GIoU 都知识考虑了两个框的重叠程度，在目标检测中，我们希望在重叠程度相同的情况下，两个框能够挨得足够近（即框的中心要尽量靠近），因此 DIoU 在 IoU 的基础上考虑了两个框的中心距离
$$
L_{DIoU} = 1- IoU + \frac{\rho^2 (b,b_{gt})}{c^2}
$$

- $\rho(b,b_{gt})$ 为预测框和真实框中心点的欧几里得距离
-  c 是覆盖两个矩形的最小外接矩形的对角线长度



# RCNN

## 具体的过程

<img src="https://img.leftover.cn/img-md/202503112203479.png" alt="image-20250311220322333" style="zoom:33%;" />

<img src="https://img.leftover.cn/img-md/202411210917942.png" alt="image-20241121091739799" style="zoom:50%;" />

主要思路：

1. 选出候选区域：

   使用**选择性搜索算法**生成一系列候选区域(anchor)，每个候选区域是一个矩形框，可能包含目标或背景，例如 2000 个候选区域

2. 对每个候选区域进行大小调整，调整为统一的大小，227*227

3. 使用 CNN 进行特征提取：

   将每个候选区域输入到 CNN 中（RCNN 使用的是 AlexNet）进行特征提取，2000 个候选区域生成一个 2000*4096 维的矩阵

4. 进行分类预测：

   将生成的 2000*4096 维的矩阵输入到 20 个 SVM 中,得到了当前每个候选区域属于每个类别的概率。（例如候选区域 1 属于 cat 的概率为 0.9，属于 car 的概率为 0.3，属于 dog 的概率为 0.4），最后得到一个 2000*20 的矩阵

   > d 因为这里是总共有 20 个类别，因此需要 20 个 SVM，SVM 是一个二分类器

5. 非极大值抑制（NMS）：

   分别对 2000*20 维的矩阵中的每一列（即每一个类别进行非极大值抑制 NMS），剔除重叠的建议框，得到该列/该类别中得分最高的一些建议框

   **NMS 的具体过程**：

   -  如下图所示，假设现在有 5 个候选区域，现在要对 car 类别进行 NMS

     <img src="https://img.leftover.cn/img-md/202411211513001.png" alt="image-20241121151309871" style="zoom:33%;" />

     1. 找出 car 类别中剩余的所有候选框中属于 car 类别最大的候选区域，假如为 C

     2. 让剩余的候选区域 A，B，D，E 分别与 B 进行 交并比（IOU）的计算，若>阈值（0.5），则剔除这些候选区域，例如这里 B、D，保留预测结果 C

     3. 再对其他类别进行上述的过程

     4. 若还有候选区域，则进行下一轮筛选，选出属于 car 类别的最大候选区域，为 A

     5. 让其他候选区域与 A 进行 IOU 计算，若>阈值（0.5），则剔除这些候选区域，例如这里 E，保留预测结果 A

        因此对于 car 类别，我们得到了 A、C 2 个候选框

6. 修正 bounding box，对 boundingbox 做微调

   这一步我们需要训练一个 bb 回归器，用来精确地调整候选框位置和尺寸的关键技术。

   > 主要目的就是将初始候选框微调至更接近真实目标的位置，从而提高检测精度

   **具体步骤：**

   - 输入：候选框的特征向量（CNN 提取的特征）、候选框的初始坐标（x,y,w,h）

   - 目标：学习与真实框的偏移量，输出下面的四个参数

     
     $$
     \bigtriangleup  x = \frac{x_{gt} - x_{proposal } }{ w_{proposal}}  , \bigtriangleup  y = \frac{y_{gt} - y_{proposal}}{ h_{proposal}} \\
      
      \bigtriangleup  w = \log \frac{w_{gt} }{w_{proposal}} , \bigtriangleup  h = \log \frac{ h_{gt} }{h_{proposal}}
     $$
     
   - 根据预测的偏移量更新候选框的位置：

   
   $$
   x_{\text{adjusted}} = x_{\text{proposal}} + \Delta x \cdot w_{\text{proposal}}\\
     
     y_{\text{adjusted}} = y_{\text{proposal}} + \Delta y \cdot h_{\text{proposal}}\\
     
     w_{\text{adjusted}} = w_{\text{proposal}} \cdot e^{\Delta w}, \quad \\
     
     h_{\text{adjusted}} = h_{\text{proposal}} \cdot e^{\Delta h}
   $$

## R-CNN 训练的过程

1. 正负样本准备，对于训练集中的所有图像，我们采样`选择性搜索`的方法来获取候选框，得到 2000 个候选区域，但不是每个候选区域都会拿去训练，只有正样本才会拿去训练

   > - 正样本：与某真实框（Ground Truth）重叠度（IoU）大于 0.5 的候选区域。
   >- 负样本：与所有真实框的 IoU 小于 0.5 的候选区域。

2. 选择预训练模型，例如 R-CNN 选择的是 AlexNet
3. 将候选区域输入进入 CNN 中进行微调
4. 将提取的特征用来训练多个 SVM，每个 SVM 都会对候选区域进行分类

5. boundingbox 回归

   如上面第六步所说，我们使用提取的特征以及真实框的位置，预测候选框的偏移量，根据偏移量调整候选框位置。

> ⚠️训练过程没有 NMS

## R-CNN 的缺点

1. 训练阶段多，步骤繁琐，需要训练 CNN，SVM，Boundingbox 回归器
2. 训练耗时：一个图片产生 2000 个候选区域，需要对每个候选区域进行 CNN 特征提取
3. 存储需求高：需要将 CNN 训练之后的特征向量保存到磁盘，5000 张图片产生几百 G 的特征文件
4. 候选区域要经过 crop/wrap 进行固定大小，从而保证输出的特征向量的长度相同，但是这样就会导致图片变形

# SPP-Net

![image-20241121232212375](https://img.leftover.cn/img-md/202411212322445.png)

SPP-net 对 R-CNN 的改进有 2 个

<img src="https://img.leftover.cn/img-md/202411212317560.png" alt="image-20241121231730507" style="zoom:50%;" />

1. **减少卷积计算：**R-CNN 是先选出候选框，再对每个框提取特征，因此由于候选框很多，训练过程很慢，而**SPP-Net** 则是先将整个图片使用 CNN 进行特征提取，得到很多个特征图，再将候选区域的坐标与特征图上的坐标进行映射，从而得到原始图片的候选区域在特征图上的位置

   > 这样我们就只需要对每张图片放到 CNN 中训练即可，从而加快了 CNN 训练的速度

   <img src="https://img.leftover.cn/img-md/202411212322791.png" alt="image-20241121232200725" style="zoom:50%;" />

   <img src="https://img.leftover.cn/img-md/202411212320728.png" alt="image-20241121232043683" style="zoom: 50%;" />

2. **防止图片内容变形：** 由于 FC 层需要保证输入数据的维度相同，而 R-CNN 是通过 crop+wrap 的方式保证候选区域的长宽相等，从而保证了输入到 FC 层的输入数据的维度相同，但是这样会导致图片的变形

   > 第一步中得出了候选区域在特征图上的位置，映射出来的特征向量大小不固定，我们将这些特征向量塞给 SPP 层（空间金字塔变化层），SPP 层接收任何大小的输入，输出固定大小的特征向量，我们将每个候选区域分别分为 1*1，2*2,4\ *4 的子图，分别对子图的每个区域做 max pooling，因此一个候选区域在一个特征图上就得 出了 16+4 +1 个数， 若有 2 56 个特征图，则 得到 21* 256 维的特征向量，从而保证了特征向量的维度相同

<img src="https://img.leftover.cn/img-md/202411212316236.png" alt="image-20241121231614092" style="zoom:50%;" />





# Fast R-CNN

![image-20250311220723444](https://img.leftover.cn/img-md/202503112207586.png)

<img src="https://img.leftover.cn/img-md/202503112207793.png" alt="image-20250311220736637" style="zoom:50%;" />

相比于 R-CNN 的改进之处：

1. 仅在整张图象上执行卷积神经网络的前向传播，而不是对每个 anchor 对区域进行 CNN

2. ROI pooling,具体过程如下

   > 会对每个候选区域（anchor）都做 ROI pooling

![image-20250311221632772](https://img.leftover.cn/img-md/202503112216940.png)

Fast R-CNN 对**SPP-Net**的改进之处：

1. **使用 ROI pooling 层 替换 SPP 层：** 简化了池化操作，进一步提高了速度，**同时也使得损失可以反向传播到卷积层**，具体来说就是：

   <img src="https://img.leftover.cn/img-md/202411220911749.png" alt="image-20241122091134677" style="zoom: 50%;" />

- SPP 层会使用`4*4 ,2*2 ,1*1`的子图分别进行`最大池化`,从而得到`21`个值，而 ROI pooling 层则只使用一个子图，将映射到 feature map 上的每个候选区域划分为`N * M` 个区域（N，M 是超参数，可以自己指定），分别对每个区域做 max pooling，得到`N*M`个值，从而保证输入到 FC 层的特征向量的长度相等，**虽然相比于 SPP 层，会损失一些精度，但是速度得到了很大的提升，并且误差可以反向传播到卷积层**

2. **实现端到端训练：**

   SPPNET 的问题：CNN 的训练和 SVM 分类器、Boundingbox 回归器的训练是分离了，二者的训练独立

3. Fast R-CNN 使用神经网络替代 SVM 分类器，并且在最后使用 Softmax 函数输出 N+1 个类别（因为可能有些候选区域没有任何物体，是背景，因此需要多一个类别）的概率

   - 使用多任务损失函数

     $L_{cls}$ 为交叉熵损失 ， $L_{bbox}$ 为 MAE（L1 损失）
     $$
     L = L_{cls} + L_{bbox}
     $$



# Faster R-CNN

Fast R-CNN 通常需要在选择性搜索中生成大量的提议区域（anchor），Faster R-CNN 则提出将选择性搜索替换为 RPN（`类似于一个小型的目标检测`），从而减少提议区域的生成数量，并保证目标检测的精度。

> Faster R-CNN 只将生成提议区域的方法从选择性搜索改为了区域提议网络，模型的其余部分保持不变。



## 具体的步骤

<img src="https://img.leftover.cn/img-md/202503112213583.png" alt="image-20250311221329514" style="zoom:33%;" />

<img src="https://img.leftover.cn/img-md/202503112213043.png" alt="image-20250311221339920" style="zoom:50%;" />

1. 通过 CNN 生成特征图
2. 之后特征图输入到 RPN 网络中，生成一些候选区域
3. 之后候选区域输入到 ROI pooling，生成固定长度的特征向量
4. 再利用特征向量输入到 FC 层，进行分类和 bounging box 回归

## RPN 网络的具体原理

<img src="https://img.leftover.cn/img-md/202411222056999.png" alt="image-20241122205628934" style="zoom: 50%;" />



1. 使用 3*3 的滑动窗口遍历特征图，padding =1 ，stride =1，这样 feature map 上的每个点都会作为滑动窗口的中心，然后每个中心点都会生成很多 anchor`（论文中是9个anchor，三种尺度[128,256,512],三种比例[1:1，1:2，2:1]），一共9种anchor`，**长度通过尺度和比例进行一一计算，得到最原始的候选区域**
2. 将每个 anchor 输入分类分支预测该 anchor 为前景 和背景的概率（使用交叉熵损失）
3. 将每个 anchor 输入回归分支预测 anchor 的偏移量，用于修正 anchor 的位置（使用 L1 损失）
4. NMS 筛选候选区域（推理阶段）：在预测得分较高的候选区域中应用 NMS，去除高度重叠的冗余候选框，仅保留最有可能包含目标的框。（通常保留每张图像的前 300 个候选区域）
5. 最后将得到的候选区域传入 ROI Pooling，输出长度相等的特征向量

## 训练策略

anchor 的正负样本的选取`（通常正负样本为1:3）`：

正样本：

- 与某个真实框（Ground Truth Box）的 IoU（Intersection over Union）大于 0.7
- 即使 IoU 小于 0.7，但如果该锚框是某个真实框的最佳匹配（IoU 最大），也会被视为正样本。这种规则确保每个真实框至少有一个对应的正样本。

负样本：

- 与所有真实框的 IoU 小于 0.3（典型的背景区域）

忽略样本(不参与训练)：

- 与真实框的 IoU 在 0.3 和 0.7 之间（这些锚框的分类标签不参与损失计算，因为它们既不是明确的前景，也不是明确的背景）

# YOLO V1

## 网络架构

<img src="https://img.leftover.cn/img-md/202411231734829.png" alt="image-20241123173450780" style="zoom:67%;" />

我们上面讲的 Faster R-CNN 通过 RPN 网络生成候选区域，Fast R-CNN 则使用选择性搜索生成候选区域，但是 YOLO 不太一样，YOLO 将目标检测问题视为一个回归问题，使用了一个统一的 CNN，24 个卷积层+2 个 FC 层，输出为一个`S×S×(B×5 +C)`的 tensor。

- S×S： 将图像划分为 S×S 的网格

- B：每个网络预测的边界框的数量（YOLO 根据经验使用两个候选框，经过算法来调整每个网络的候选框的大小）

  > V1 版本使用 2 个候选框，太少了，这也是之后改进的一个方向

- 5:每个边界框的参数`（x,y,w,h,confidence）`

  > confidence 在训练阶段和推理阶段的设置的方式不同

- C: 数据集中类别的个数

## 损失函数

![image-20241123114758312](https://img.leftover.cn/img-md/202411231147422.png)

- 位置损失
  $$
  \text{Localization Loss} = \lambda_{\text{coord}} \sum_{i=0}^{S^2} \sum_{j=0}^{B} \mathbb{1}_{ij}^{\text{obj}} \left[ (x_i - \hat{x}_i)^2 + (y_i - \hat{y}_i)^2 + (\sqrt{w_i} - \sqrt{\hat{w}_i})^2 + (\sqrt{h_i} - \sqrt{\hat{h}_i})^2 \right]
  $$

  - **λ_coord:** 通常设置为 5，用于强调精确定位的重要性

  - w,h 要开根号的原因：

    > 现实中物体有大有小，而大的物体可能误差变化一点点，损失就会变的很大，小的物体可能偏差很大，但是他的损失很小,取平方根可以使得大目标和小目标在损失贡献上均等

- 置信度损失
  $$
  \text{Confidence Loss} = \sum_{i=0}^{S^2} \sum_{j=0}^{B} \left[ \mathbb{1}{ij}^{obj} (C_i - \hat{C}i)^2 + \lambda{noobj} \mathbb{1}{ij}^{noobj}\quad (C_i - \hat{C}_i)^2 \right]
  $$

  - **λ_noobj:** 通常设置为 0.5，以降低背景区域的权重，减少对背景区域的惩罚

- 分类损失
  $$
  \text{Classification Loss} = \sum_{i=0}^{S^2} \mathbb{1}{i}^{obj} \sum_{c \in classes} (p_i(c) - \hat{p}_i(c))^2
  $$

  > $\mathbb{1}{i}^{obj}$ 表示 cell i 中是否有物体

## 推理过程

YOLO v1 接受一张图像作为输入，经过网络处理后，输出 S×S×(B×5 + C) 的张量

1. 根据置信度阈值（例如 0.5），筛选出可能包含目标的边界框
2. NMS：对于同一个类别，先选择置信度得分最高的边界框，再去除与该边界框 IOU 值比较高的其他边界框，重复此过程
3. 最后生成最终的边界框和对应的类别标签



## 缺点

1. 每个网格只能预测一个类别，如果一个网络中有很多小物体，例如鸟群，则无法准确预测
2. 由于 YOLO 是从训练数据中学习，来预测边界框，**因此他很难推广到 新的或者不寻常的纵横比的物体**
3. YOLO1 使用的损失函数在对待大物体和小物体时的处理方式一样，没有区别对待，但是在实际中，一个大的边界框，一个小的错误通常不会造成太大影响，但是在一个小的边界框中，一个小的错误对 IOU 的影响要大很多



# YOLO V2

## V2 相对于 V1 的改进

1. 移除了 FC 层，基于 Googlenet 进行修改，得出了一个新的网络（Darknet）`[有 19 个卷积层和 5 个 maxpooling 层]`，**移除了 FC 层，使用 BN 来加快收敛速度，并且有正则化的效果**

2. V1 训练时的图像是 224×224 的，而 V2 使用了更大的分辨率的图像进行训练（使用了多尺度训练）

3. **K-Means 聚类得出先验 anchor：**

   - 在 YOLO V1 中，每个网络使用 2 个 anchor，Faster R-CNN 使用了 9 个 anchor，但是这些 anchor 都是根据经验人工挑选的，网络通过学习来调整 anchor 的中心点和大小，但如果我们一开始就为网络选择更好的先验，就能让网络更容易学会预测良好的检测结果。
   - 在训练集上使用 K-means 聚类，如下图所示，k=5 时可以在模型的复杂度和 recall 中做出很好的权衡

   > 因此这里我们选择 5 个 anchor，每个 anchor 的长宽为当前类别的长宽的均值

   ![image-20241123170321487](https://img.leftover.cn/img-md/202411231703632.png)



4. **限制中心点的预测的位置：**

   - 我们以前对于边界框的中心 x,y 的训练大致如下：

     1. 利用回归模型得到中心点的偏移值 $t_x$、 $t_y$ 

     2. 根据 $t_x$、 $t_y$ 进行修正,再与真实的边界框计算误差

        > 例如，如果预测值为 t_x=1 ，则框会向右移动锚点框的宽度，如果预测值为 t_x=−1 ，则框会向左移动相同的宽度。
        >
        >这种方法不受约束，因此任何锚点框都可以出现在图像中的任何一点上，而与预测锚点框的位置无关。在随机初始化的情况下，模型需要很长时间才能稳定下来，预测出合理的偏移量

        ![image-20241123171316828](https://img.leftover.cn/img-md/202411231713888.png)

   - 现在：对中心位置的预测进行了限制，中心点只能在当前网格内进行偏移
    ![image-20241123171814049](https://img.leftover.cn/img-md/202411231718113.png)



5. **对所得到的特征进行微调（特征融合）：**

   在 YOLO V1 的版本中说到，V1 版本对于小目标的检测比较差，而 YOLO V2 使用了 DarkNet 得到了 13×13 的特征图，其感受野很大，对于大型物体的预测已经足够了，但是小物体需要更精确的定位，而 YOLO V2 将上一层的特征图进行变化成与最后一层的特征图的尺寸相同，如下图所示：将 26×26×512 的特征图变化为 4×13×13×512 的特征图，并且与最后一个 13×13×1024 的特征图进行融合，得到 13×13×3072 的特征图，这样可以提升其对于小物体的检测精度

   ![image-20241123172831271](https://img.leftover.cn/img-md/202411231728322.png)

6. **多尺度训练：**

   由于模型没有 FC 层，因此我们可以随便改变图像的输入大小，因此 YOLO V2 使用了不同尺寸的图像对模型进行训练，每 10 个 epochs 就随机选择新的图像尺寸,从{320,352,……,608} 中选择，然后继续训练，因此 V2 版本可以在各种输入维度上进行预测

   > 相比于没有多尺度训练，MPA 提升了 1.8



## 特点

1. 多尺度训练
2. 使用 K-means 得到先验 anchor
3. 特征融合提升小目标的检测能力（但是提升有限）
4. 对中心点点预测进行限制，使模型的训练更加稳定
5. 移除了 FC 层和 Dropout，添加 BN，使得模型的训练更快收敛

# YOLO V3

## 特点

1. 多尺度预测，使用不同的特征图对大、中、小物体进行预测
2. 在类别预测中，使用逻辑回归替代 Softmax，因为一个 box（物体） 可能包含多个类别，例如（人、女人）
3. 网络结构改进，引入残差连接的思想，去除 FC、池化层，使用 stride =2 进行下采样
4. 使用逻辑回归输出置信度的预测值，而不是使用 IOU，具体原因请看[使用 YOLO V2 中的使用 IOU 作为置信度有何不好？](# 使用 YOLO V2 中的使用 IOU 作为置信度有何不好？)



## 核心网络架构

![image-20241124100707681](https://img.leftover.cn/img-md/202411241007813.png)

在 YOLO V3 中改进了网络的结构，命名为 Darknet53,使用了 ResNet 的思想，**引入残差连接，提升了网络的层数，没有池化层和 FC 层，全部都是卷积，下采样通过 stride =2 来实现**

## 输出的含义

以 13×13×3×（4+1+80）为例：

- 13×13 为特征图的大小
- 3 表示每个网格 3 个 anchor box
- 4 表示 x,y,h,w
- 1 表示置信度
- 80 表示当前数据集的 80 个类别

## Bounding Box Prediction 的改进

V3 版本中使用逻辑回归法为每个 box 预测置信度得分，若某个先验框与真实框的 IOU 最大，则这个先验框的置信度为 1，若先验框与真实框的 IOU 不是最大，但是超过了某个阈值（0.5），则忽略该样本。

> 应该由与 ground truth 最大的先验框负责拟合该 ground truth

### 使用 YOLO V2 中的使用 IOU 作为置信度有何不好？

1. 很多预测框与 ground truth 的 IOU 最高只有 0.7，而不是 1，因此模型的学习能力一直提升不上去

2. coco 中小目标的 IOU 对像素的偏移很敏感，无法有效地学习

## 多尺度预测（提升小目标的检测能力）

在 YOLO V3 中，如下图所示

- 最后输出的特征图为 13×13×1024 的，这个特征图的感受野比较大，用来预测大物体
- 将最后一层的特征图进行 2 倍上采样，与上一层的特征图进行融合得到 26×26×256 的特征图，用来预测中等大小的物体
- 再将得到的 26×26×256 的特征图进行 2 倍上采样与上一层的特征图融合，得到 52×52×128 的特征图，用来预测小物体

<img src="https://img.leftover.cn/img-md/202411241013413.png" alt="image-20241124101350325" style="zoom: 50%;" />

### 3 种 scale

在 V3 中，每个网络有 3 个先验框，我们会使用 3 种不同大小的特征图进行预测，因此一共有 9 种先验框，和 V2 类似，使用聚类的方式得到 9 种 anchor，再将其分为 3 类，分别用于预测大、中、小物体,例如在 coco 数据集上得到了这 9 种先验框：(10×13),(16×30),(33×23),(30×61),(62×45),(59×119),(116×90),(156×198),(373×326)

## Softmax 层替代

在物体检测任务中，一个物体可能存在多个重叠的标签，例如（女人和人），而 Softmax 方法需要假设每个 anchor box 只有一个类别。**在 YOLOV3 中，使用逻辑回归来预测每个类别的概率，损失函数使用二元交叉熵损失**

## CIOU 对 boundingbox 回归

在 yolo v1 中 boingingbox 对损失函数为（使用的是 MSE）:
$$
\text{Localization Loss} = \lambda_{\text{coord}} \sum_{i=0}^{S^2} \sum_{j=0}^{B} \mathbb{1}_{ij}^{\text{obj}} \left[ (x_i - \hat{x}_i)^2 + (y_i - \hat{y}_i)^2 + (\sqrt{w_i} - \sqrt{\hat{w}_i})^2 + (\sqrt{h_i} - \sqrt{\hat{h}_i})^2 \right]
$$
而 yolo v4 中使用了 CIOU 作为 bounding box 的损失：


$$
\text{CIoU} =1- \text{IoU} + \frac{\rho^2(\mathbf{b}, \mathbf{b}^{gt})}{c^2} + \alpha v
$$

- $\rho(\mathbf{b}, \mathbf{b}^{gt})$ 是预测边界框$ \mathbf{b}$ 和真实边界框 $\mathbf{b}^{gt}$ 的中心点之间的欧氏距离。
- c 是包含预测和真实边界框的最小闭合边界框的对角线长度。
- v 是用于衡量预测边界框与真实边界框在长宽比（Aspect Ratio）上的一致性的项

![image-20241124164918693](https://img.leftover.cn/img-md/202411241649781.png)



## DIOU-NMS

### Soft-NMS

- 最开始的 NMS 中，会去除跟最高置信度框有高度重叠的其他框，这种方法在处理一些密集/重叠的目标的时候常常会导致漏检。

- 而 Soft-NMS 不直接删除与最高置信度框有高度重叠的其他框，而是通过降低这些框的置信度分数来保留它们，根据调整后的置信度分数重新排序候选框。

### DIOU NMS

DIOU—NMS 在决定是否删除一个框的时候，不仅仅只考虑 IOU，**因为传统 NMS 在处理密集目标或高度重叠的目标时，可能会因仅基于 IoU 而误删有效的候选框。**

1. 选择置信度最高的框
2. 计算其他框与这个框的（$IoU - R_{DIoU(M,B_i)}$） 的值，若这个值**大于**阈值 $\varepsilon $ ，则删除这个框,若小于这个阈值，则表示两个框离得很近，可能是两个物体有重合，不删除这个框

$$
R_{DIoU} = \frac{\rho ^2(M,B_i)}{c^2}
$$

- $\rho (M,B_i)$ 表示 两个框中心点的欧式距离
- c ：包含框 M 和框 $B_i$ 的最小闭合框的对角线长度

## SAM（Spatial attention module）空间注意力模块

### CBAM（**Convolutional Block Attention Module**）

CBAM 由通道注意力模块 、 空间注意力模块组成

1. **通道注意力模块：**

   **通道注意力模块**旨在通过赋予不同通道不同的权重，突出对当前任务更有用的特征通道，抑制不相关或冗余的通道信息。

   - 对输入的特征图的每个通道进行全局最大池化和平均池化，每个通道会得到两个特征向量，
   - 将特征向量输入到一个共享的感知机（包含一个隐藏层和一个输出层）得到输出，
   - 每个通道会得到两个特征向量，将两个特征向量相加，通过 sigmoid 函数得到最后的通道注意力权重

   ![image-20241124200840339](https://img.leftover.cn/img-md/202411242008462.png)

2. **空间注意力模块：**

   **空间注意力模块**旨在通过赋予不同空间位置不同的权重，突出图像中更为重要的区域，抑制背景或不相关的区域。

   - 使用通道维度上的平均池化和最大池化，得到两个不同的特征图
   - 将两个特征图沿通道维度进行拼接，形成一个新的特征图
   - 将拼接后的特征图输入到卷积层（7×7 卷积），得到新的特征图
   - 最后通过 sigmoid 函数得到最后的空间权重

   ![image-20241124200853573](https://img.leftover.cn/img-md/202411242008689.png)

### yolo v4 中的 SAM

Yolo v4 对原本的 SAM 做了改进，不对输入特征图进行最大池化和平均池化，而是直接进行卷积得到一个特征图，再将其输入到 sigmoid 函数中，得到最后的空间权重，这样会更快

![image-20241124200822192](https://img.leftover.cn/img-md/202411242008310.png)





## SPP +PAN

在 yolo v3 中，提出了数据融合，将上层的特征图通过上采样与下层的特征图进行融合

而 yolo v4 中提出了一种 **既将上层的特征图通过上采样与下层的特征图进行融合，也将下层的特征图通过下采样与上层的特征图融合 **，从而提高检测的准确率

<img src="https://img.leftover.cn/img-md/202411241937492.png" alt="image-20241124193740381" style="zoom:50%;" />

同时 yolo v4 使用了 spp 增大感受野，用于检测大目标

## CSPNet

yolo v4 中引入 CSPNet 来`提高模型速度`

1. 在深层网络中，随着层数的增加，信息的传递和梯度的反向传播变得更加困难，容易导致特征重复、计算冗余以及梯度消失等问题。这些问题不仅降低了模型的训练效率，还限制了模型的性能提升。

**原理:**

- 将输入的特征图按照一定比例（例如一半）分为 2 部分：part1，part2

- part2 经过一系列的卷积，残差连接进行特征提取和处理

- part1 则直接传递到后续的卷积层或残差块

- 最后将 part1 和 part2 进行 concat，形成融合之后的特征图,继续传递到网络的后续层

  ![image-20241124202227757](https://img.leftover.cn/img-md/202411242022818.png)



## Mish

使用了 Mish 损失函数来替换 Relu，会增加计算量，精确度有些许提升

## 标签分配方法

### YOLO X 的标签分配方式

YOLO v7 的标签分配 集成了 YOLO X 和 YOLO V5 的长处，YOLO X 是 anchor free 的，使用的是[simOTA 的标签分配方式](# simOTA（动态标签分配） )。

### YOLO V5 的标签分配方式

而 YOLO v5 是 anchor-based 的: 通过 k-means 聚类得到 9 种先验的 anchor 框，并不是每个 GT 都会使用 9 个 anchor 框的，通过计算 9 个 anchor 框的长宽比、GT 的长宽比，若这两个值相差小于某个阈值，则该 GT 会使用这个 anchor。

> YOLo v5 训练的时候定义正负样本：看 GT 的中心落在哪个 grid 上，则取这个 grid，除此之外，会将每个 grid 分为 4 象限，假设中心点落在第四象限，如下图所示，则还会选取右边和下边的 grid。
>
>理论上每个 GT 会取 3 个 grid，则 3 个 grid 作为上面匹配该 GT 的 anchor 的中心，则会得到 3-27 个正样本。

<img src="https://img.leftover.cn/img-md/202412022106128.png" alt="image-20241202210658990" style="zoom:33%;" />



### YOLO V7 的标签分配方式

YOLO v7 则是基于 simOTA 的标签分配方式，在第一步筛选候选正样本的时候使用了 YOLO V5 的标签分配方式来筛选正样本，保留了 simOTA 的剩余步骤进一步筛选正样本。

YOLO V7 中有 aux_head 和 lead_ head,aux_head 作为辅助，`主要用于提高recall，防止漏检`，其筛选正样本的策略和 lead_head 相同，不过条件更佳宽松。而 lead_head 就是使用上面的步骤进行筛选的。



**aux_head 和 lead_head 筛选正样本的步骤的不同：**

- 第一步筛选候选正样本的时候，lead_head 会选取 GT 中心点对应的 grid 以及与之接近的两个 grid 对应的预测框作为正样本，而 aux_head 则是选择 GT 中心点对应的 grid 以及 与之接近的 4 个 grid 对应的预测框作为正样本（即上下左右 4 个 grid）。

  <img src="https://img.leftover.cn/img-md/202412022119879.png" alt="image-20241202211948789" style="zoom: 50%;" />

- simOTA 部分，lead_head 是选择候选正样本与 GT 的 IOU 最大的 10 个 IOU 进行求和并取整； 而 aux_head 则是选择最大的 20 个 IOU 进行求和并取整







# anchor 的生成和多尺度检测

- anchor 的生成

```python

def multibox_prior(data, sizes, ratios):
    """生成以每个像素为中心具有不同形状的锚框"""
    """
    
    :param data: feature map 
    :param sizes:  一个数组，元素表示anchor的面积是特征图的面积的几倍,[0.25,0.50.75]
    :param ratios: 宽高比：[1,0.5,2]
    :return:  1, h*w* n+m-1,4 的tensor
    [x_min,y_min,x_max,y_max] 
    """
    in_height, in_width = data.shape[-2:]
    device, num_sizes, num_ratios = data.device, len(sizes), len(ratios)
    # 每个像素生成的anchor的数量
    boxes_per_pixel = (num_sizes + num_ratios - 1)
    size_tensor = torch.tensor(sizes, device=device)
    ratio_tensor = torch.tensor(ratios, device=device)

    # 为了将锚点移动到像素的中心，需要设置偏移量。
    # 因为一个像素的高为1且宽为1，我们选择偏移我们的中心0.5
    offset_h, offset_w = 0.5, 0.5
    steps_h = 1.0 / in_height  # 在y轴上缩放步长
    steps_w = 1.0 / in_width  # 在x轴上缩放步长

    # 生成锚框的所有中心点
    center_h = (torch.arange(in_height, device=device) + offset_h) * steps_h
    center_w = (torch.arange(in_width, device=device) + offset_w) * steps_w
    shift_y, shift_x = torch.meshgrid(center_h, center_w, indexing='ij')
    shift_y, shift_x = shift_y.reshape(-1), shift_x.reshape(-1)

    # 生成“boxes_per_pixel”个高和宽，其高宽是相对于输入的h，w来的，是一个0-1的数
    # 之后用于创建锚框的四角坐标(xmin,xmax,ymin,ymax)
    w = torch.cat((size_tensor * torch.sqrt(ratio_tensor[0]),
                   sizes[0] * torch.sqrt(ratio_tensor[1:]))) \
        * in_height / in_width  # 处理矩形输入
    h = torch.cat((size_tensor / torch.sqrt(ratio_tensor[0]),
                   sizes[0] / torch.sqrt(ratio_tensor[1:])))
    # 除以2来获得半高和半宽
    # 之后用中心点的坐标+ (-w,-h,w,h) 就可以得到（x_min,y_min,x_max,y_max）
    anchor_manipulations = torch.stack((-w, -h, w, h)).T.repeat(
        in_height * in_width, 1) / 2

    # 每个中心点都将有“boxes_per_pixel”个锚框，
    # 所以生成含所有锚框中心的网格，重复了“boxes_per_pixel”次
    out_grid = torch.stack([shift_x, shift_y, shift_x, shift_y],
                           dim=1).repeat_interleave(boxes_per_pixel, dim=0)
    output = out_grid + anchor_manipulations
    return output.unsqueeze(0)
```

# SSD 的实现



## 模型的结构

SSD 有 5 个模块组成，即 5 个 stage，每个 stage 之后都会生成 anchor，进行预测，最后把所有 stage 生成的 anchor 和 预测的结果合并起来一起计算损失

```python
def get_blk(i):

    if i == 0:
        blk = base_net()
    elif i == 1:
        blk = down_sample_blk(64, 128)
    elif i == 4:
        blk = nn.AdaptiveMaxPool2d((1, 1))
    else:
        blk = down_sample_blk(128, 128)
    return blk
```



## 损失

对每个 anchor 都会计算分类损失，背景类不会计算回归损失，最终的 loss = cls_loss + bbox_loss

```python
def calc_loss(cls_preds, cls_labels, bbox_preds, bbox_labels, bbox_masks):
    """

    :param cls_preds:  [batch_size ,所有的anchor数量，类别数+1]
    :param cls_labels:
    :param bbox_preds: [batch_size ,所有的anchor数量 *4]
    :param bbox_labels:
    :param bbox_masks:
    :return:
    """
    # num_classes包括了背景
    batch_size, num_classes = cls_preds.shape[0], cls_preds.shape[2]
    cls_loss = nn.CrossEntropyLoss(reduction='none')
    bbox_loss = nn.L1Loss(reduction='none')
    # 得到的是[batch,1] 求出的是每个样本的分类损失
    cls = cls_loss(cls_preds.reshape(-1, num_classes), cls_labels.reshape(-1)).reshape(batch_size, -1).mean(dim=1)
    bbox = bbox_loss(bbox_preds * bbox_masks, bbox_labels * bbox_masks).mean(dim=1)
    return cls + bbox
```



## 训练过程

1. 通过 CNN 得到每个 stage 生成的 anchor 和 anchor 的预测类别和 offset

2. 得到了 anchor 之后，要为 anchor 分配 ground_truth （大概是计算所有的 anchor 和 每个 ground-truth 的 IOU，只会保留 IOU > 阈值的 anchor，其他的会被分为背景类，如果某个 anchor 和多个 gt 都 IOU 都很高，则只会被分配给最高的 IOU 的 gt），这里就可以得到所有 anchor 的类别。

   之后再计算非背景类的 anchor 与 gt 之间 的距离，得到所有 anchor 的 offset

3. 之后根据 bbox_labels, bbox_masks, cls_labels 和  cls_preds 、bbox_preds 计算损失，再反向传播更新权重

```python
def train():
    batch_size = 32
    train_iter, _ = d2l.load_data_bananas(batch_size)
    device, net = d2l.try_gpu(), TinySSD(num_classes=1)
    trainer = torch.optim.SGD(net.parameters(), lr=0.2, weight_decay=5e-4)
    num_epochs, timer = 20, d2l.Timer()
    animator = d2l.Animator(xlabel='epoch', xlim=[1, num_epochs],
                            legend=['class error', 'bbox mae'])
    net = net.to(device)
    for epoch in range(num_epochs):
        # 训练精确度的和，训练精确度的和中的示例数
        # 绝对误差的和，绝对误差的和中的示例数
        metric = d2l.Accumulator(4)
        net.train()
        for features, target in train_iter:
            timer.start()
            trainer.zero_grad()
            X, Y = features.to(device), target.to(device)
            # 得到多尺度的锚框，以及锚框预测类别和偏移量
            anchors, cls_preds, bbox_preds = net(X)
            # 为每个锚框标注类别和偏移量
            # bbox_masks 即表示这个anchor是否是背景类，背景类为0，否则为1，背景类不计算回归损失
            bbox_labels, bbox_masks, cls_labels = d2l.multibox_target(anchors, Y)
            # 根据类别和偏移量的预测和标注值计算损失函数
            l = calc_loss(cls_preds, cls_labels, bbox_preds, bbox_labels,
                          bbox_masks)
            l.mean().backward()
            trainer.step()
            metric.add(cls_eval(cls_preds, cls_labels), cls_labels.numel(),
                       bbox_eval(bbox_preds, bbox_labels, bbox_masks),
                       bbox_labels.numel())
        cls_err, bbox_mae = 1 - metric[0] / metric[1], metric[2] / metric[3]
        animator.add(epoch + 1, (cls_err, bbox_mae))
    print(f'class err {cls_err:.2e}, bbox mae {bbox_mae:.2e}')
    print(f'{len(train_iter.dataset) / timer.stop():.1f} examples/sec on '
          f'{str(device)}')
```

## 预测

1. 同样将图片输入进 net，得到多个尺度的 anchor 以及这些 anchor 的类别预测 和 bbox 预测。

2. 对类别预测做一个 softmax，然后找到每个 anchor 类别预测的最大值，这个就是预测的分类结果，最后根据预测的分类结果、bbox_preds,anchor 进行后处理（nms）。

3. 最后剩下的那些 anchor 就是最终的结果，然后根据 bbox_preds 和 anchor 的位置将其可视化出来。

```python
def predict(X):
    net.eval()
    anchors, cls_preds, bbox_preds = net(X.to("cpu"))

    output = d2l.multibox_detection(cls_probs, bbox_preds, anchors)
    # 取出没有被nms掉的的anchor
    idx = [i for i, row in enumerate(output[0]) if row[0] != -1]
    return output[0, idx]
```
