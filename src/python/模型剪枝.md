# 剪枝概述



## 模型剪枝定义

   模型剪枝也叫模型稀疏化，跟模型量化不同，模型量化是对每一个/部分权重参数进行压缩，即将 FP32 变为 INT8 等，稀疏化则是尝试直接“删除”部分权重参数。模型剪枝低原理是通过剔除不重要的权重，使得模型减少参数量和计算量，从而加快推理速度，尽量保证模型的精度不受影响。`其实模型中的参数是存在冗余的，剔除少量不权重的权重可以一定地起到正则化的效果（类似于dropout），从而提高模型的精度，但随着剪枝的数量越来越多，模型的精度就会逐渐下降`



> 通常来说卷积层的冗余性比 FC 层更少

## 结构化剪枝 vs 非结构化剪枝

### 非结构化剪枝

对独立的权重或神经元连接进行剪枝。非结构化剪枝并不关心权重在网络中的位置，只是根据某种标准（例如，权重的绝对值大小）来决定是否移除这个权重

移除权重后，剩下的权重分布是稀疏的，即大多数权重为零。非结构化剪枝能极大降低模型的参数量和理论计算量。

> 但是现有硬件架构的计算方式无法对其进行加速，通常需要特殊的硬件或软件支持来有效利用结果模型的稀疏性。所以在实际运行速度上得不到提升，需要设计特定的硬件才可能加速。

### 结构化剪枝

这种剪枝方法可能涉及到移除整个神经元、卷积核、层或者更复杂的结构。通常以 filter 或者整个网络层为基本单位进行剪枝。一个 filter 被剪枝，那么其前一个特征图和下一个特征图都会发生相应的变化，但是模型的结构却没有被破坏，仍然能够通过 GPU 或其他硬件来加速，因此这类方法被称之为结构化剪枝



> 非结构化剪枝关注于移除单个不重要的权重，导致权重矩阵变得稀疏，而结构化剪枝关注于移除整个神经单元或过滤器，导致网络结构上的变化，比如减小层数或通道数。在实际应用中，结构化剪枝通常能够提供更好的性能和压缩比，因为它能够更好地利用模型的内在结构。然而，结构化剪枝的实现通常更加复杂，需要更多的计算资源和对模型的深入理解。`非结构化剪枝则相对简单，但可能不会达到结构化剪枝的压缩效果。因为非结构化剪枝需要硬件的支持才能可能加速`

## 剪枝的时机

1. **训练后剪枝：**训练模型 -> 剪枝 -> 对剪枝后的模型微调 -> 迭代剪枝（重复剪枝和微调的过程）
2. **训练时剪枝：**

   - 训练一定的轮次，让模型的权重初步学习到有意义的模式

   - 在每个 epoch 结束，根据剪枝算法进行剪枝（例如重要性评估），将对应的权重设置置为 0

   - 在剪枝完了之后，继续训练一个 epoch，通过反向传播，运行之前被修剪过的权重更新

   - 重复上述过程，直到模型收敛或者达到预定的训练 epochs
3. **训练前剪枝：**彩票假说认为，一个随机初始化、密集的网络中，包含一个子网络，在经过相同的迭代次数之后，能够达到后原始网络相近的准确度。在彩票假设中，将“中奖”的子网络重置为网络最初的权重后重新训练。最后得到的结果可以追上甚至超过原始的稠密网络

> **那么如何找到这个子网络呢？**
>
>目前常用的方式一般是在初始化阶段采用随机剪枝的方法。随机选择的连接被修剪，并且该过程重复多次以创建各种稀疏网络架构。这背后的想法是，如果在训练之前以多种方式进行修剪，可能就能够跳过寻找彩票的过程。

# 神经网络的稀疏性

### 权重稀疏

大多数神经网络中，我们对卷积层 和 全连接层的权重数值进行直方图统计可以发现，**权重数组分布接近于正态分布**,且越接近于 0，权重越多。

> `而权重数值的绝对值可以看做权重重要性的度量，绝对值越大则越重要，反之则不重要，因此权重存在很多冗余。`

<img src="https://img.leftover.cn/img-md/202504121906059.png" alt="image-20250412190620810" style="zoom:33%;" />



### 激活稀疏

现代的神经网络中通常会使用 Relu 激活函数，该激活函数会令<0 的输入都产生 0 的输出，因此通过了 relu 激活函数之后，网络的输出中存在很多 0，因此 xxx 通过统计，CNN 中很多神经元的激活值都很低，因此我们可以将这些`零神经元`移除，从而减少网络的参数量

> 由于移除的是神经元，因此其属于结构化剪枝





### 梯度稀疏

很多网络由于参数量很大，都会采取分布式的训练方式，通常采用分布式随机梯度下降（Distributed SGD）算法：允许 N 台节点共同完成梯度更新的后向传播训练任务。其中每台主机均保存一份完整的参数拷贝，并负责其中 1N 参数的更新计算任务。按照一定时间间隔，节点在网络上发布自身更新的梯度，并获取其他 N−1 台节点发布的梯度计算结果，从而更新本地的参数拷贝。

很显然，随着参与训练任务节点数目的增多，网络上传输的模型梯度数据量也急剧增加，网络通信所占据的资源开销将逐渐超过梯度计算本身所消耗的资源，从而严重影响大规模分布式训练的效率。

因此为了克服分布式训练中的通信瓶颈，梯度稀疏（`Gradient Sparsification`）得到了广泛的研究

- 选择固定比例的正负梯度进行更新
- 在网络上仅仅传输那些绝对值超过阈值的梯度（梯度越大则越重要）





# 稀疏性约束

## 权重的 L1 /L2 正则化

通过在损失函数中添加 L1/L2 正则化可以让一些不重要的权重趋向于 0，这样会导致训练后的神经网络有更好的稀疏性，从而实现更高的压缩比

## BN 层γ因子的 L1/L2 正则化

在下面的基于基于 BN 层的缩放因子γ的剪枝方法中，通过 BN 层的γ的绝对值大小来对通道进行剪枝。和 L1/L2 正则化方法类似，`对于此类方法，通常会在训练的时候给损失函数添加一个有关γ的正则化，让一些不重要的channel的对应的 γ-> 0, 这样训练后的网络有更好的稀疏性。`

根据下面的第一张图可以得出，$\lambda$ 参数的选择对于剪枝和模型的精度有很大的影响。$\lambda $ 太小，则训练后的网络稀疏性不足，不能够实现很高的压缩比。$\lambda $ 太大，则会导致损失函数中正则化的权重占比太多，从而影响模型的精度。合适的 $\lambda$ 可以保证模型的精度不下降，并且训练后的模型有很好的稀疏性，从而实现更高的压缩比。



第一张图和第二张图对比可以看出，当模型没有添加正则化约束的时候，模型也有一定的稀疏性，但是不多，随着剪枝率的提升，模型的精度出现大幅的下降。

# 剪枝标准

## L1 范数

使用 L1 范数的大小来评估权重的重要性，值越大，则表示越重要



## L2 范数

使用 L2 范数的大小来评估权重的重要性，值越大，则表示越重要

## 基于梯度的大小

核心思想：在训练过程中，`权重的梯度反映了权重对输出损失函数的影响程度`，梯度的绝对值越大则表示权重对输出损失的影响也越大。那么那些梯度的绝对值很小的权重就可以被裁剪掉来减小模型的规模



## 基于 BN 层的缩放因子γ

Network Slimming 提出了一种基于尺度(Scaling-based)的剪枝方法。他会识别并剪枝那些对模型输出影响不大的整个通道（即一组特征映射），而不是单个权重，即根据 BN 层中的缩放因子（γ）的绝对值大小来判断哪些通道重要，哪些不重要。`在训练过程中通过在损失函数中添加一个关于 γ 的L1正则化来鼓励 γ 趋向于0 （可以实现更好的稀疏性）`。这样的话，不重要的通道的缩放因子γ 将变得非常小，从而在训练之后将这些通道剪枝。







# 剪枝频率

通常包含迭代式剪枝和单次剪枝

### 迭代式剪枝

是一个渐进式的模型剪枝方法，即重复地执行 `剪枝-> 微调->评估` 的步骤，这个过程中逐步削减模型中的权重，而不是一次性剪除大量的权重。

### 单次剪枝

单次剪枝即在训练完之后对模型进行一次性的剪枝操作，根据某种剪枝标准来确定哪些权重可以被移除。

单次剪枝会极大地受到噪声的影响，而迭代式剪枝方法则会好很多，因为它在每次迭代之后只会删除掉少量的权重，然后周而复始地进行其他轮的评估和删除，这就能够在一定程度上减少噪声对于整个剪枝过程的影响.

**但是对于大模型来说，微调的成本很高，所以更倾向于使用单次剪枝的方法**
