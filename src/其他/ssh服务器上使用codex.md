我有一台国内的云服务器，平常是 ssh 连接服务器远程开发，我有 chatgpt 的会员，因此想在远程服务器上使用 codex，但由于网络原因，国内的网络使用不了 codex



## 前置要求
本地需要有科学上网

## 如何通过登录的验证
下载完 codex 之后通常会需要你登陆验证。

1. 我们在本地下载 codex，完成验证之后
2. 之后在你的 home 目录下会有一个.codex 文件夹，即`~/.codex`
3. 复制这个文件夹到服务器上的`~/.codex`就可以通过验证了，之后服务器上就可以绕过浏览器登录这一步了

## 终端使用 codex
通过了登录之后，这时候还是用不了 codex，发消息的时候会一直转圈圈

我们需要配置代理来让 ssh 的服务器使用本地的代理



1. ssh 连接的命令中加入-R 参数

```bash
ssh -R 7890:127.0.0.1:7890 user@remote_server
```

上面的-R 参数的作用是将远程的 7890 端口转发到本地 7890

> 我本地的梯子使用的端口是 7890，你本地的梯子如果是别的端口，则配置别的端口，这里根据实际的配置即可
>

vscode 中只需要在连接的配置文件中加入`RemoteForward 7890 127.0.0.1:7890`即可

```bash
# 在你的 ~/.ssh/config 文件中
Host my-remote-server
  HostName <服务器IP或域名>
  User <用户名>
  # 添加下面这行
  RemoteForward 7890 127.0.0.1:7890
```

2. 配置环境变量

这里有一些需要注意的点，

> 端口号的配置是和你上面的配置有关的，上面我们配置的是将服务器的 7890 端口转发到本地的 7890 端口，因此下面的端口配置的是服务器的 7890
>

> 两个都是[http://127.0.0.1:7890](http://127.0.0.1:7890)，即无论是 http 还是 https 的流量都会转发到服务器的[http://127.0.0.1:7890](http://127.0.0.1:7890)，
>
> 不建议`https_proxy="https://127.0.0.1:7890"`，这里会有问题，而且比较麻烦
>

```bash
# 根据你的本地代理类型设置
# 如果是 HTTP/HTTPS 代理
export http_proxy="http://127.0.0.1:7890"
export https_proxy="http://127.0.0.1:7890"

# 如果是 SOCKS5 代理
#export http_proxy="socks5://127.0.0.1:8080"
#export https_proxy="socks5://127.0.0.1:8080"
#export all_proxy="socks5://127.0.0.1:8080"
```

之后再`source ~/.bashrc`使环境变量生效



3. 测试你的代理是否生效了

我们先回顾一下上面的配置，我们首先将服务器上的所有的流量转发到了服务器的 7890 端口，由于我们在 ssh 的时候配置了端口转发，服务器上的 7890 端口的流量会转发到本地的 7890 端口，这样我们 ssh 连接的服务器就可以科学上网了。

**分别**运行下面两个命令测试你的出口 ip 地址，**如果返回的 ip 地址不是你自己的服务器的 ip**，则说明成功了

```bash
curl ip.sb
```

```bash
curl https://api.ipify.org
```

4. 之后我们就在终端打开 codex，正常使用即可
